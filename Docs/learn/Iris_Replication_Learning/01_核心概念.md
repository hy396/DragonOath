# 01. Iris 核心概念

## Iris 是什么

**Iris**（Iris Replication System）是 UE 的新一代网络复制架构，目标是用更统一、可扩展、可并行化的管线替代传统 **Generic** 复制路径（基于 `AActor` Channel、`IsNetRelevantFor` 等）。

在 UE 5.8 中：

- Iris 核心在引擎模块 **IrisCore**（`Engine/Source/Runtime/Net/Iris`），随引擎编译，不依赖单独安装。
- 默认 **未启用**：`net.Iris.UseIrisReplication` 默认为 `0`，仍走 Generic。
- 启用后，同一套 `AActor` / `UObject` 游戏代码大多可继续工作，但底层走 `UReplicationSystem` + **Bridge** 桥接。

## 与传统 Generic 复制的主要差异

| 维度 | Generic（传统） | Iris |
|------|-----------------|------|
| 核心对象 | Actor Channel | NetObject + ReplicationState |
| 相关性 | `IsNetRelevantFor`、RepGraph 等 | **NetObjectFilter**（如 Spatial、None） |
| 优先级 | `GetNetPriority` | **NetObjectPrioritizer** |
| 序列化 | 属性/RPC 传统路径 | **NetSerializer** + ReplicationState 描述符 |
| 脏标记 | 轮询 `NetUpdateFrequency` 等 | 倾向 **Push Model**（显式标脏） |
| 扩展点 | ReplicationGraph 节点 | Bridge 配置、Filter、Prioritizer、DataStream |

Iris 不是「换一个 CVar 就完事」——Filter/Prioritizer/Serializer 配置会实质影响行为与带宽。

## 心智模型

```text
  游戏世界（AActor / UActorComponent / UObject）
                    |
            UEngineReplicationBridge
         （引擎侧 UObjectReplicationBridge 实现）
                    |
         注册为 NetObject，绑定 Filter / Prioritizer
                    |
            UReplicationSystem
                    |
    +---------------+----------------+
    |                                |
 NetObjectFilter              NetObjectPrioritizer
 (谁该看到？)                  (先同步谁？带宽不够裁谁？)
    |                                |
    +---------------+----------------+
                    |
         ReplicationDataStream / NetBlob
                    |
              UNetDriver → 客户端
```

**Bridge** 负责在「游戏对象」与「复制系统内部 NetObject」之间翻译生命周期：生成、销毁、Filter 分配、停止复制原因等。

## 关键类型

### UReplicationSystem

复制系统中枢。管理 NetObject 注册、连接视图、Filter/Prioritizer 句柄、DataStream 等。

源码：`Iris/ReplicationSystem/ReplicationSystem.h`

### UObjectReplicationBridge / UEngineReplicationBridge

- `UObjectReplicationBridge`：抽象桥，连接游戏与 `UReplicationSystem`
- `UEngineReplicationBridge`：引擎内置实现，处理 `AActor`、`UActorComponent` 等

注册 Actor 时可传 `FActorReplicationParams`，指定 Filter 策略：

- `ConfigFilter` — 按 `ObjectReplicationBridgeConfig::FilterConfigs` 自动分配
- `AlwaysRelevant` — 动态 Filter 为 None（始终相关）
- `DefaultSpatial` — 使用默认空间 Filter（引擎默认名 `Spatial`）
- `ExplicitFilter` — 指定 `ExplicitDynamicFilterName`

### NetObjectFilter

决定某个 NetObject 对哪些连接可见。引擎默认在 `BaseEngine.ini` 注册：

- `Spatial` → `NetObjectGridWorldLocFilter`（网格 + 世界位置）
- `NotRouted` → `FilterOutNetObjectFilter`（不复制）

`DynamicFilterName=None` 表示**始终相关**（无动态 Filter）。

### NetObjectPrioritizer

在带宽有限时决定同步顺序与裁剪。默认示例：

- `DefaultPrioritizer` → `SphereNetObjectPrioritizer`
- `PlayerStatePrioritizer` → `NetObjectCountLimiter`

### NetSerializer / ReplicationState

Iris 用描述符 + 序列化器表达「要复制哪些成员、如何编码」。与 Push Model、结构体 `NetSerialize` 兼容性相关，进阶见后续 `06_PushModel与NetSerializer.md`（待写）。

### DataStream

数据流通道，如 `Replication`、`NetToken`。在 `BaseEngine.ini` 的 `DataStreamDefinitions` 中声明。

## 启用判定链（简化）

引擎在创建 `UNetDriver` 时综合以下因素决定是否用 Iris（见 `UEngine::WillNetDriverUseIris`）：

```text
1. IrisNetDriverConfigs：该 NetDriver 是否 bCanUseIris=true
2. net.Iris.UseIrisReplication / 引擎默认
3. GameInstance / GameMode 对 ReplicationSystem 的偏好覆盖
4. 命令行 -UseIrisReplication=0|1
```

**三者都允许**才会真正初始化 Iris 复制。

## 对 DragonOath 的启示

横版 2.5D 联机：

- **Spatial Filter** 可基于世界位置做网格剔除，但横版视野可能是「横向带宽」而非圆形距离——后期可能要自定义 Filter 或调 Grid 参数。
- 玩家 Pawn、Boss、队友状态、局部弹幕可能需要不同 Filter / Prioritizer 组合。
- GAS 相关结构体在 `BaseEngine.ini` 已有 `SupportsStructNetSerializerList` 示例（如 `GameplayCueParameters`），接入 GAS 时需关注 Serializer 兼容性。
