# 00. Iris 复制系统学习路线

## 学习目标

学完本系列后，你应能：

1. 说明 Iris 与传统 Generic 复制在架构上的差异。
2. 解释 `UReplicationSystem`、`UObjectReplicationBridge`、`UEngineReplicationBridge` 各自职责。
3. 在 DragonOath 中通过 ini / CVar / 命令行启用 Iris，并完成 PIE 多人验证。
4. 为横版 2.5D 联机场景配置基础的 **Filter**（相关性）与 **Prioritizer**（优先级）。
5. 说清楚 Iris 与 ReplicationGraph 的选型关系（见专题 04）。

## 前置知识

- UE 联机基础：Server / Client、`AActor` 复制、`UNetDriver`
- 建议了解 Push Model 概念（`MARK_PROPERTY_DIRTY_FROM_NAME` 等）
- 建议先阅读 [02_Technical_Architecture.md](../../02_Technical_Architecture.md) 联机章节
- 可与 [ReplicationGraph 学习](../ReplicationGraph_Learning/README.md) 对照阅读

## 阶段划分

### 阶段 A：概念（1～2 天）

- [ ] 阅读 `ReplicationSystem.h`、`EngineReplicationBridge.h` 公开 API
- [ ] 完成 [01_核心概念.md](01_核心概念.md)
- [ ] 画出「UObject → Bridge 注册 → Filter → Prioritizer → DataStream 发送」流程图

### 阶段 B：启用与配置（2～3 天）

- [ ] 阅读 `BaseEngine.ini` 中 Iris 配置段
- [ ] 完成 [02_引擎配置与启用.md](02_引擎配置与启用.md)
- [ ] 在实验分支用 `-UseIrisReplication=1` 或 `net.Iris.UseIrisReplication=1` 跑通 PIE

### 阶段 C：Filter / Prioritizer（2～4 天）

- [ ] 完成 [03_Filter与Prioritizer.md](03_Filter与Prioritizer.md)
- [ ] 为 `Info`、默认 Pawn、测试 Actor 验证 Spatial / None / NotRouted 行为

### 阶段 D：DragonOath 实践（3～5 天）

- [ ] 按 [05_DragonOath实践计划.md](05_DragonOath实践计划.md) 落地
