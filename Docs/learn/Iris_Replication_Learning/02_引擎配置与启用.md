# 02. 引擎配置与启用

## 默认状态（UE 5.8）

根据 `IrisConfig.cpp`：

```cpp
// CVar 默认值
net.Iris.UseIrisReplication = 0   // 0 = 使用传统 Generic 复制
```

即：**不额外配置时，项目走传统复制**。Iris 能力已编译进引擎，但需显式打开。

## 启用方式一览

| 方式 | 示例 | 说明 |
|------|------|------|
| CVar | `net.Iris.UseIrisReplication 1` | 运行时切换；PIE 控制台可测 |
| 命令行 | `-UseIrisReplication=1` | 启动参数，优先级高 |
| ini | `IrisNetDriverConfigs` + 引擎段覆盖 | 按 NetDriver 细粒度控制 |

命令行在 `IrisCoreModule` 启动早期解析，可覆盖 CVar 默认值。

## 步骤 1：允许 GameNetDriver 使用 Iris

引擎 `BaseEngine.ini` 已包含：

```ini
+IrisNetDriverConfigs=(NetDriverDefinition=GameNetDriver, bCanUseIris=true)
```

DragonOath 可在 `Config/DefaultEngine.ini` **追加或确认**（若被覆盖）：

```ini
[/Script/Engine.Engine]
+IrisNetDriverConfigs=(NetDriverDefinition=GameNetDriver, bCanUseIris=true)
```

其他 NetDriver 示例：

```ini
; 按名称
+IrisNetDriverConfigs=(NetDriverName=DemoNetDriver, bCanUseIris=false)

; 通配符
+IrisNetDriverConfigs=(NetDriverWildcardName="SecondaryNetDriver*", bCanUseIris=true)
```

`bCanUseIris=false` 时，即使全局开了 Iris，该 Driver 也不会用。

## 步骤 2：打开全局 Iris 开关

**方式 A — 项目 ini（推荐实验阶段）**

```ini
[ConsoleVariables]
net.Iris.UseIrisReplication=1
```

**方式 B — 编辑器额外启动参数**

```text
-UseIrisReplication=1
```

**方式 C — PIE 控制台**

```text
net.Iris.UseIrisReplication 1
```

然后重新进入 PIE（是否热切换有效取决于 NetDriver 初始化时机，**建议重启 PIE**）。

## 步骤 3：验证是否生效

Listen Server PIE 后检查：

1. **日志**：搜索 `Iris`、`IsUsingIrisReplication`、`forcing ... to use the Iris replication system`
2. **代码/蓝图**：若有访问 `UNetDriver`，可查 `IsUsingIrisReplication()`
3. **对比实验**：同一地图分别用 `0` / `1` 跑 `stat net`

若配置了 `bCanUseIris=true` 但 CVar 为 0，仍会走 Generic。

## 步骤 4：复制 Bridge 与 Channel 配置

引擎默认（`BaseEngine.ini`）：

```ini
[/Script/IrisCore.ObjectReplicationBridgeConfig]
RequiredNetDriverChannelClassName=/Script/Engine.DataStreamChannel
DefaultSpatialFilterName=Spatial
```

项目一般**先继承引擎默认**，除非要改默认空间 Filter 名或 Channel 类。

Filter / Prioritizer 的**定义**在：

```ini
[/Script/IrisCore.NetObjectFilterDefinitions]
[/Script/IrisCore.NetObjectPrioritizerDefinitions]
```

**按类分配**在：

```ini
[/Script/IrisCore.ObjectReplicationBridgeConfig]
+FilterConfigs=(...)
+PrioritizerConfigs=(...)
```

## 步骤 5：GameInstance / GameMode 覆盖（了解即可）

`UEngine::WillNetDriverUseIris` 还会读取：

- `UGameInstance::GetDesiredReplicationSystem()`
- `AGameModeBase::GameNetDriverReplicationSystem`

游戏可在 C++ 层强制某一局使用 Iris 或 Generic。DragonOath 第一期可不改，仅用 ini + CVar。

## 实验插件 Iris.uplugin

路径：`Engine/Plugins/Experimental/Iris/`

该插件 **EnabledByDefault: false**，且 IrisCore 已在引擎 Runtime 内。DragonOath 实践以 **启用 Iris 复制 + 配置** 为主，**不必**为了学习而强制启用 Experimental 插件，除非后续文档明确要求其附加模块。

## 常见问题

| 现象 | 可能原因 |
|------|----------|
| 开了 CVar 仍是 Generic | `IrisNetDriverConfigs` 中 `bCanUseIris=false` 或未配置 GameNetDriver |
| 部分 Actor 不复制 | `FilterConfigs` 设为 `NotRouted`，或 Spatial 距离外 |
| 与 RepGraph 同时配 | RepGraph 的 `ReplicationDriverClassName` 作用于 **Generic** 路径；Iris 启用后不走 RepGraph（见 04） |
| PIE 行为不一致 | 检查 `-UseIrisReplication` 与 `net.Iris.UseIrisReplication` 是否冲突 |
