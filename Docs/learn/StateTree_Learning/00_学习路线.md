# 00. StateTree 学习路线

## 学习目标

学完本系列后，你应能：

1. 说明 StateTree 在 UE 行为系统中的定位（vs. BehaviorTree、蓝图、C++ 状态机）。
2. 解释 State、Transition、Condition、Task、Evaluator、Event 各自作用与调用时机。
3. 在 Actor 上正确配置 `UStateTreeComponent`，绑定属性，跑通最小状态循环。
4. 使用 StateTree 编辑器：创建资产、编辑节点、绑定参数、调试运行时状态。
5. 为 DragonOath 怪物 AI 设计合理的行为树结构（索敌、靠近、攻击、返回、死亡）。

## 前置知识

- UE 基础：`AActor`、组件、Tick、DataAsset
- 建议已了解 **GAS** 基础（DragonOath 怪物属性与技能用 GAS）
- 建议已阅读 [02_Technical_Architecture.md](../../02_Technical_Architecture.md) 中「怪物 AI」章节
- BehaviorTree 经验可选（StateTree 概念部分对应，但不是前置条件）

## 阶段划分

### 阶段 A：概念入门（1～2 天）

- [ ] 阅读 `StateTree.h`、`StateTreeTypes.h`、`StateTreeTaskBase.h` 公开 API
- [ ] 完成 [01_核心概念.md](01_核心概念.md)
- [ ] 画出「State → Transition(Condition) → State」循环图

### 阶段 B：Actor 集成（2～3 天）

- [ ] 完成 [02_在Actor上使用.md](02_在Actor上使用.md)
- [ ] 创建 `UStateTreeComponent` 最小示例：Idle → Action → Idle
- [ ] 理解 Schema 的作用：约束可用的 Task/Condition/Evaluator
- [ ] 掌握属性绑定：StateTree 如何读写 Actor/Component 数据

### 阶段 C：编辑器与调试（1～2 天）

- [ ] 完成 [03_状态树编辑器.md](03_状态树编辑器.md)
- [ ] 在 PIE 中打开 StateTree 调试器，观察状态切换与 Task 执行
- [ ] 使用日志 (`LogStateTree`) 和断点排查行为

### 阶段 D：内置节点与选型（2～3 天）

- [ ] 完成 [04_内置Task与Condition.md](04_内置Task与Condition.md)
- [ ] 对照引擎 `StateTreeModule/Public/Conditions/` 和 `Tasks/` 目录
- [ ] 为「索敌→靠近→攻击→返回」选型合适的 Task/Condition

### 阶段 E：DragonOath 落地（按 M5 里程碑）

- [ ] 完成 [05_DragonOath实践计划.md](05_DragonOath实践计划.md)
- [ ] 设计 `ST_Enemy_Melee`（近战杂兵）完整行为树
- [ ] 设计 `ST_Enemy_Ranged`（远程怪）完整行为树
- [ ] 与 GAS 集成：StateTree Task 触发 GameplayAbility

## 实践清单（可勾选）

```text
[ ] 启用 StateTree 插件（UE 5.8 默认已启用）
[ ] 创建 UStateTree 资产，选择 Schema（Actor 用默认，Mass 用 MassStateTreeSchema）
[ ] 在 EnemyCharacter 上添加 UStateTreeComponent，绑定 StateTree 资产
[ ] 编辑最小状态树：Root → Idle → (Condition: HasTarget) → Attack → Idle
[ ] PIE 验证状态切换正确
[ ] 添加 StateTree 事件：OnDamaged → Stun/Death
[ ] 为 Attack 状态绑定动画 Montage 与 GAS Ability 触发
[ ] 调试：观察 ActiveStates、InstanceData、Transition 日志
```
