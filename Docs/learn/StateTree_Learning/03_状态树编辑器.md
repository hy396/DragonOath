# 03. 状态树编辑器与调试

## 编辑器概述

StateTree 编辑器是 UE5 内置的资产编辑器，用于可视化编辑状态机。打开方式：

1. 在 Content Browser 创建资产 → Miscellaneous → StateTree
2. 双击 `ST_xxx` 资产即可打开

### 编辑器布局

```text
┌──────────────────────────────────────────────────┐
│  工具栏：Compile | Save | Debug | Settings       │
├───────────────────────┬──────────────────────────┤
│                       │                          │
│   状态树图            │   细节面板               │
│   (Graph Editor)      │   (Details Panel)        │
│                       │                          │
│   [Root]              │   选中节点后显示属性：    │
│   ├─ [State] Idle     │   - Name                 │
│   ├─ [State] Combat   │   - Conditions           │
│   └─ [State] Dead     │   - Tasks                │
│                       │   - Transitions          │
│                       │   - Property Bindings    │
│                       │                          │
├───────────────────────┴──────────────────────────┤
│  编译日志 / 调试面板                              │
└──────────────────────────────────────────────────┘
```

## 基本操作

### 创建 StateTree 资产

1. Content Browser → 右键 → **Miscellaneous → StateTree**
2. 创建时选择 Schema（Actor 默认用 `StateTreeComponentSchema`）
3. 打开资产，看到默认 [Root] 节点

### 添加状态节点

- 右键 [Root] → **Add State** → 选择一个 State 类型
- 或者拖拽节点从左侧模板面板
- 支持的类型：
  - **State**：标准状态
  - **Group State**：可包含子状态的选择器（Sequence / Selector）
  - **Linked State**：引用另一个 StateTree 资产

### 添加 Transition

- 从状态节点边缘拖出连线到目标节点
- 在 Transition 线上右键 → **Add Condition**
- Transition 方向由箭头指示

### 添加 Task / Condition / Evaluator

- 选中状态 → 细节面板 → **Tasks** 区域 → 点击 `+` 添加
- 选中 Transition → 细节面板 → **Conditions** 区域 → 点击 `+` 添加
- 选中状态 → 细节面板 → **Evaluators** 区域 → 点击 `+` 添加

### 属性绑定

点击属性旁的「绑定」按钮（链接图标），弹出绑定面板：

- **Bound Property**：选择绑定的目标数据结构
- **Copy Type**：复制时机（On Enter / On Tick / On Event 等）
- **Binding Expression**：简单的绑定表达式

## 调试

### 启用调试

1. PIE 运行中
2. 打开 StateTree 资产编辑器
3. 工具栏或 Viewport 中选择目标 Actor
4. 编辑器右上角切换 **Debug Mode**

### 调试视图

调试模式下：

- **当前 Active State**：绿色高亮
- **上一个 Active State**：橙色
- **Transition 条件**：满足的显示绿色，不满足显示灰色
- **Task 状态**：Running（蓝）/ Succeeded（绿）/ Failed（红）

### 断点

在 Task 或 Condition 节点上右键 → **Toggle Breakpoint**

- 当 StateTree 执行到该节点时暂停
- 可在细节面板检查 Instance Data 当前值

### LogStateTree

控制台命令：

```
Log LogStateTree Verbose          # 启用详细日志
Log LogStateTree Warning           # 只显示警告
StateTree.Debug.EnableCategory All # 启用所有调试类别
```

关键日志类别：

| 类别 | 作用 |
|------|------|
| `LogStateTree` | 核心执行日志 |
| `LogStateTree.OnEnterState` | 进入状态时打印 |
| `LogStateTree.OnExitState` | 离开状态时打印 |
| `LogStateTree.OnTick` | Tick 日志（量大） |
| `LogStateTree.OnTransition` | Transition 触发时打印 |

### Gameplay Debugger

启用 Gameplay Debugger（`'` 键），按数字键切换到 **StateTree** 分类（如已集成），显示：

- 当前 Active States 名称
- 当前正在执行的 Task
- Instance Data 摘要

## 常见陷阱

### 1. Schema 不匹配

创建 StateTree 资产时选错了 Schema → 可用的 Task/Condition 不对 → **新建资产重选 Schema**（Schema 创建后不可更改）。

### 2. 属性绑定路径错误

绑定属性时引用了不存在的字段或路径 → 编译时会有 Warning → 检查绑定路径拼写和结构体层级。

### 3. Task 忘记返回 Running

```cpp
// 错误：Tick 中直接返回 Succeeded，状态立即结束
virtual EStateTreeRunStatus Tick(...) const override
{
    DoSomething();
    return EStateTreeRunStatus::Succeeded;  // 只执行一帧！
}

// 正确：完成后再返回 Succeeded
virtual EStateTreeRunStatus Tick(...) const override
{
    if (IsDone())
        return EStateTreeRunStatus::Succeeded;
    DoSomething();
    return EStateTreeRunStatus::Running;
}
```

### 4. EnterState 中做耗时操作

`EnterState` 在 Transition 过程中同步调用，不应放耗时逻辑（如 `MoveTo`）。改为：

- `EnterState`：标记开始（如设置目标点）
- `Tick`：逐帧推进（如 `MoveTowardTarget`）
- 抵达后返回 `Succeeded`

### 5. 循环 Transition

A → B → A 没有有效 Condition 保护 → 无限循环切换 → 加上合理的 Delay 或退出条件。

## 编辑器效率技巧

1. **Ctrl+Click** 节点可快速预览其 Task/Condition 列表
2. **右键 → Collapse State** 将子状态折叠为 Linked Asset（复用）
3. **Ctrl+Shift+F** 在 StateTree 中搜索节点名称
4. 右键 → **Copy State** 可以复制整个状态及其子节点
5. 编译错误会在工具栏以红色标记，点击可跳转到问题节点

## 练习

1. 创建 `ST_Test_Debug`，包含 Idle → Move → Idle 循环。
2. PIE 中打开 StateTree 调试器，观察 Idle 和 Move 之间的切换。
3. 在 `Tick` 中故意不返回 Running，观察行为差异。
4. 启用 `Log LogStateTree Verbose`，阅读输出日志。
