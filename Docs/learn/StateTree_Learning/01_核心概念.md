# 01. StateTree 核心概念

## StateTree 解决什么问题

在 UE 中描述"行为逻辑"有多种方式：

| 方案 | 适用场景 | 限制 |
|------|----------|------|
| 蓝图脚本 | 简单顺序逻辑 | 状态切换复杂时难维护 |
| C++ 状态机 | 性能敏感、逻辑固定 | 迭代慢、非设计师友好 |
| BehaviorTree | 传统 AI 行为 | 事件驱动弱、与 GAS 集成不自然 |
| **StateTree** | 事件驱动 + 状态化 AI + GAS 集成 | 学习曲线中等 |

**StateTree** 是 UE5 引入的层次化状态机框架，核心设计理念：

> 用 **State** 描述角色当前状态，用 **Transition** + **Condition** 控制切换，用 **Task** 执行具体逻辑，用 **Evaluator** 提供 Tick 级评估，用 **Event** 响应外部信号。

## 核心六要素

```
                          ┌─────────────┐
                    ┌─────│  Evaluator  │─────┐
                    │     │  (每帧评估)  │     │
                    │     └─────────────┘     │
                    ▼                         ▼
  ┌──────────┐  Transition  ┌──────────┐  Transition  ┌──────────┐
  │  State A │──────────────▶│  State B │──────────────▶│  State C │
  │  Idle    │  + Condition  │  Combat  │  + Condition  │  Dead    │
  │          │◀──────────────│          │               │          │
  └──────────┘  Transition   └──────────┘               └──────────┘
       │                          │
       ▼                          ▼
  ┌──────────┐              ┌──────────┐
  │  Tasks   │              │  Tasks   │
  │  (逻辑)  │              │  (逻辑)  │
  └──────────┘              └──────────┘
```

### 1. State（状态）

角色的当前状态节点，如 `Idle`、`Combat`、`Patrol`、`Dead`。

- **Root State**：树的根节点，每个 StateTree 必须有一个根
- **Child State**：子状态，父状态激活时子状态才可能激活
- **Active State**：当前正在执行的状态（可以有多个并行）
- **State Type**：`State`（标准）、`Group`（分组/选择器）、`Linked`（引用其他 StateTree）

### 2. Transition（过渡）

状态之间的有向连线，定义了从 A 到 B 的切换规则。

- **Trigger**：事件触发（如 `OnDamaged`）
- **Priority**：多条 Transition 同时满足时的优先级
- **Delay**：条件满足后延迟多久再切换
- **Bidirectional**：是否可反向（如 A ↔ B）

```cpp
// 源码：EStateTreeTransitionType
enum class EStateTreeTransitionType : uint8
{
    None,         // 禁用
    Succeeded,    // 当前状态 Succeeded 时自动切换
    Failed,       // 当前状态 Failed 时自动切换
    Next,         // 顺序执行（无条件）
    NextSelectable, // 下一个可选状态
    GotoState,    // 带 Conditions 的条件跳转
};
```

### 3. Condition（条件）

附加在 Transition 上的判断逻辑，返回 `true`/`false`。

- **Enter Conditions**：进入该状态的前提条件（放在 State 上）
- **Transition Conditions**：Transition 的触发条件
- 引擎内置条件见 [04_内置Task与Condition.md](04_内置Task与Condition.md)

### 4. Task（任务）

在 Active State 中执行的具体逻辑。生命周期：

```
EnterState() ──► Tick() ──► Tick() ──► ... ──► ExitState()
                    │
                    └── 返回 Running → 继续 Tick
                    └── 返回 Succeeded/Failed → 触发 Transition
```

关键 API（来自 `FStateTreeTaskBase`）：

```cpp
virtual EStateTreeRunStatus EnterState(FStateTreeExecutionContext& Context, const FStateTreeTransitionResult& Transition) const;
virtual void ExitState(FStateTreeExecutionContext& Context, const FStateTreeTransitionResult& Transition) const;
virtual EStateTreeRunStatus Tick(FStateTreeExecutionContext& Context, const float DeltaTime) const;
virtual void StateCompleted(FStateTreeExecutionContext& Context, const EStateTreeRunStatus CompletionStatus, const FStateTreeActiveStates& CompletedActiveStates) const;
```

### 5. Evaluator（评估器）

每帧 Tick 评估，通常用于持续监测条件（类似 BehaviorTree 的 Service）。

- 在 State 激活期间持续运行
- 可用于更新数据、检查阈值、触发事件
- 不改变状态，只提供数据给 Task/Condition 使用

### 6. Event（事件）

外部信号通过 StateTree Event 传入：

```cpp
// C++ 侧发送事件
FStateTreeEvent Event(EStateTreeEventType::External, MyEventTag);
StateTreeComponent->SendStateTreeEvent(Event);
```

- 可触发 Transition（如 `OnDamaged` → `Stun`）
- 可传递给 Task 处理
- 支持 `FGameplayTag` 作为事件标识

## 执行模型

```
每帧:
  1. 处理外部 Event（SendStateTreeEvent）
  2. 执行当前 Active States 的 Evaluator.Tick()
  3. 检查 Transition Conditions → 决定是否切换
  4. 对离开的状态调用 Task.ExitState()
  5. 对进入的状态调用 Task.EnterState()
  6. 执行新 Active States 的 Task.Tick()

RunStatus:
  Running   → 状态继续，下帧再来
  Succeeded → 状态完成，触发 Succeeded 类型 Transition
  Failed    → 状态失败，触发 Failed 类型 Transition
```

## 与 BehaviorTree 的概念对照

| BehaviorTree 概念 | StateTree 对应 | 差异 |
|-------------------|---------------|------|
| Composite (Selector/Sequence) | State (Group 类型) | StateTree 更偏状态思维 |
| Task Node | Task | StateTree Task 有 Enter/Tick/Exit 生命周期 |
| Decorator | Enter Condition + Evaluator | StateTree 分离了前置条件和持续评估 |
| Service | Evaluator | 类似，Evaluator 范围更灵活 |
| Blackboard | Instance Data + Bindings | StateTree 用结构体替代 Key-Value 黑板 |
| Event | StateTree Event | 都支持外部事件驱动 |

## 关键数据流

```
  Actor / Component
       │
       ▼
  ┌─────────────────────┐
  │  UStateTreeComponent │  ◄── 持有 → UStateTree 资产
  │  - InstanceData      │  ◄── 运行时状态数据（结构体）
  │  - Context           │  ◄── 执行上下文
  └─────────┬───────────┘
            │
   ┌────────┼────────┐
   │ Bind   │        │
   ▼        ▼        ▼
 Property  Actor   World
 Bindings  Data    Subsystems
```

StateTree 通过 **Property Bindings** 读写外部数据，而非"通过引用访问黑板 Key"。
