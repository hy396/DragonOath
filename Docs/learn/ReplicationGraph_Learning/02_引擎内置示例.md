# 02. 引擎内置示例：BasicReplicationGraph

## 定位

`UBasicReplicationGraph` 是 Epic 提供的**最小可运行**实现，注释明确说明：

- 仅支持 `NetCullDistanceSquared`、`bAlwaysRelevant`、`bOnlyRelevantToOwner`
- 这些标志**不能在运行时按 Actor 动态切换**
- 用于学习；复杂项目需要自定义子类

源码位置：

```text
D:\UE_5.8\Engine\Plugins\Runtime\ReplicationGraph\
  Source\Public\BasicReplicationGraph.h
  Source\Private\BasicReplicationGraph.cpp
```

## 类结构一览

`UBasicReplicationGraph` 持有三个关键成员：

| 成员 | 类型 | 用途 |
|------|------|------|
| `GridNode` | `UReplicationGraphNode_GridSpatialization2D` | 按距离/网格做空间化复制 |
| `AlwaysRelevantNode` | `UReplicationGraphNode_ActorList` | 始终对所有客户端相关 |
| `AlwaysRelevantForConnectionList` | 连接 → `AlwaysRelevant_ForConnection` 节点映射 | Owner 专属等按连接相关 |

另外还有 `ActorsWithoutNetConnection`：在生成时尚未有关联连接、但标记为仅 Owner 相关的 Actor 暂存区。

## 初始化流程

### InitGlobalActorClassSettings

为各类 Actor 在 `GlobalActorReplicationInfoMap` 中登记默认策略（如默认剔除距离、复制频率）。  
**阅读重点**：哪些类被特殊处理，默认值从哪里来（`AActor` 还是项目配置）。

### InitGlobalGraphNodes

```text
CreateNewNode<UReplicationGraphNode_GridSpatialization2D>()  → GridNode
CreateNewNode<UReplicationGraphNode_ActorList>()           → AlwaysRelevantNode
AddGlobalGraphNode(...)
```

全局 Node 对所有连接可见；连接级 Node 在 `InitConnectionGraphNodes` 中创建。

### InitConnectionGraphNodes

为每个 `UNetReplicationGraphConnection` 创建 `UReplicationGraphNode_AlwaysRelevant_ForConnection`，并记入 `AlwaysRelevantForConnectionList`。  
典型用途：玩家 Pawn、仅 Owner 可见的 UI Actor 等。

## Actor 路由：RouteAddNetworkActorToNodes

这是实践时最常改写的函数。Basic 实现的逻辑分支大致为：

```text
if (bAlwaysRelevant)
    → AlwaysRelevantNode
else if (bOnlyRelevantToOwner)
    → 对应连接的 AlwaysRelevant_ForConnection（或暂存到 ActorsWithoutNetConnection）
else
    → GridNode（空间化）
```

**练习**：打开 `BasicReplicationGraph.cpp`，对照上述分支抄一张流程图，并标注每个 `NotifyAddNetworkActor` 调用。

## 移除与重命名

- `RouteRemoveNetworkActorToNodes`：从各 Node 移除 Actor
- `RouteRenameNetworkActorToNodes`：动态 Outer 变更时更新 Node 内引用（需配置 `Net.RepGraph.HandleDynamicActorRename`）

横版项目中若大量使用 Level Streaming 或子关卡，需关注 Rename 路径。

## 仅用 ini 启用（实验用）

在**独立测试工程**或 DragonOath 的实验分支中，可先不改 C++，仅验证插件与驱动切换：

1. `.uproject` 启用插件 `ReplicationGraph`
2. `Config/DefaultEngine.ini` 增加：

```ini
[/Script/OnlineSubsystemUtils.IpNetDriver]
ReplicationDriverClassName="/Script/ReplicationGraph.BasicReplicationGraph"
```

3. PIE 多人运行，观察默认 Pawn 与放置 Actor 的复制是否正常

若连接失败或无任何复制，检查：

- 插件是否编译进目标（Editor / Game）
- 是否使用了正确的 NetDriver 段（Listen Server 与 Dedicated Server 配置可能不同）

## 与 Shooter / Lyra 的关系

引擎注释提到 ShooterGame 的 `UShooterReplicationGraph` 作为更完整范例；Lyra 也基于 ReplicationGraph 做大规模联机优化。

DragonOath 当前未包含这些样本工程。学习路径建议：

1. 先吃透 `BasicReplicationGraph`
2. 需要时再对照 Lyra 源码（若本地有 Samples）或 Epic 文档中的 Shooter 范例

## 调试建议

启用后可在控制台尝试：

```text
Net.RepGraph.PrintGraph
Net.RepGraph.PrintAll
```

配合 `stat net`、`net.PackageMap` 等观察通道数量变化。具体输出格式以 UE 5.8 为准，实验时把有效输出贴到本文件「实验记录」小节。
