# 00. ReplicationGraph 学习路线

## 学习目标

学完本系列后，你应能：

1. 说明 ReplicationGraph 解决什么问题，以及它与 `AActor::IsNetRelevantFor` 默认路径的差异。
2. 读懂并修改 `UBasicReplicationGraph` 的节点路由逻辑。
3. 在 DragonOath 中启用 ReplicationGraph 驱动，并完成一次 PIE 多人验证。
4. 为横版 2.5D 联机场景设计初步的空间化 / 始终相关 / 按连接相关策略。

## 前置知识

- UE 联机基础：Server / Client、`AActor` 复制、`NetDriver`、`UNetConnection`
- C++ 模块与 `.uproject` 插件启用
- 建议先阅读项目 [02_Technical_Architecture.md](../../02_Technical_Architecture.md) 中的联机相关章节

## 阶段划分

### 阶段 A：概念（1～2 天）

- [ ] 阅读 `ReplicationGraph.h` 文件头注释（引擎自带的高层说明）
- [ ] 完成 [01_核心概念.md](01_核心概念.md)
- [ ] 画出「Actor 生成 → RouteAdd → Node 列表 → 每连接 Gather → 通道更新」流程图

### 阶段 B：引擎示例（2～3 天）

- [ ] 阅读 `BasicReplicationGraph.h/.cpp`
- [ ] 完成 [02_引擎内置示例.md](02_引擎内置示例.md)
- [ ] 在测试工程中仅通过 ini 切换到 `BasicReplicationGraph`，对比默认复制行为

### 阶段 C：DragonOath 实践（3～5 天）

- [ ] 按 [03_DragonOath实践计划.md](03_DragonOath实践计划.md) 启用插件与模块
- [ ] 实现 `UDragonOathReplicationGraph`（可先继承 `UBasicReplicationGraph`）
- [ ] PIE 2 客户端验证：距离剔除、Owner 相关、Always Relevant 三类 Actor
- [ ] 使用 `Net.RepGraph.*` 调试命令观察节点与列表

### 阶段 D：项目化策略（持续）

- [ ] 横版关卡：按 X 轴带宽或自定义 Grid 做 2D 空间化
- [ ] 玩家、队友、Boss、弹幕/投射物分别定义复制节点策略
- [ ] 将最终策略写入 `02_Technical_Architecture.md`

## 实践清单（可勾选）

```text
[ ] 在 .uproject 启用 ReplicationGraph 插件
[ ] 新建 DragonOathReplicationGraph 模块（或放入现有 Game 模块）
[ ] 配置 DefaultEngine.ini 的 ReplicationDriverClassName
[ ] 创建测试 Actor：AlwaysRelevant / DistanceCull / OwnerOnly
[ ] PIE Listen Server + 2 Clients 验证
[ ] 记录带宽与 Actor 通道数量对比（可选：stat net）
```

## 常用控制台命令（备忘）

```text
Net.RepGraph.PrintGraph                  # 打印图结构
Net.RepGraph.PrintAll                    # 打印全局 Actor 信息
Net.RepGraph.PrintAllActorInfo           # 单 Actor 详情（需指定）
Net.RepGraph.ForceBuild                  # 强制重建图
```

具体参数以引擎 `ReplicationGraphDebugging.cpp` 为准；实践时把有效命令补充到后续调试笔记。
