# 02. 插件栈与依赖

## 总览

MassAI 不是单模块，而是一组 **Experimental** 插件链。UE 5.8 主要组成：

```text
MassEntity（引擎 Runtime 模块，非独立 .uproject 插件）
    ↑
MassGameplay（Spawner / Movement / LOD / Representation / Replication）
    ↑
MassAI（AI Behavior / Navigation / StateTree / AI Replication）
    ↑
MassCrowd（可选示例：人群沿 ZoneGraph 移动）
```

## MassAI.uplugin 模块

路径：`D:\UE_5.8\Engine\Plugins\AI\MassAI\MassAI.uplugin`

| 模块 | 类型 | 职责 |
|------|------|------|
| MassNavigation | Runtime | 通用导航 Fragment / Processor |
| MassNavMeshNavigation | Runtime | NavMesh 路径跟随 |
| MassZoneGraphNavigation | Runtime | ZoneGraph 通道导航 |
| MassNavigationEditor | Editor | 导航相关编辑器 |
| MassAIBehavior | Runtime | StateTree、LookAt、SmartObject 行为 |
| MassAIBehaviorEditor | Editor | 行为编辑器扩展 |
| MassAIReplication | Runtime | AI 相关 Mass 复制 |
| MassAIDebug | Runtime | 调试可视化 |
| MassAITestSuite | UncookedOnly | 测试 |

`EnabledByDefault: false`，需在项目中显式启用。

## MassAI 硬依赖插件

`MassAI.uplugin` 的 `Plugins` 段：

```json
MassGameplay, ZoneGraph, ZoneGraphAnnotations, SmartObjects, StateTree, NavCorridor
```

启用 MassAI 时，这些依赖通常会自动连带启用（以编辑器解析为准）。**ZoneGraph** 偏大规模室外/车道式导航；横版关卡可能更多用 **NavMesh** 或自定义 Movement Processor。

## MassGameplay.uplugin 模块（摘要）

| 模块 | 职责 |
|------|------|
| MassSpawner | `AMassSpawner`、Entity Config、Trait 基类 |
| MassMovement | 移动 Fragment / Processor |
| MassLOD | 距离 LOD，降低远距 Entity 更新频率 |
| MassRepresentation | 将 Entity 同步到 ISM、Static Mesh、Actor 代理 |
| MassReplication | Mass 实体联机复制基础 |
| MassSimulation | 模拟阶段管理 |
| MassSmartObjects | 与 SmartObjects 插件集成 |

## 在 DragonOath 中启用（实验）

在 `DragonOath.uproject` 的 `Plugins` 数组添加（实验分支）：

```json
{ "Name": "MassGameplay", "Enabled": true },
{ "Name": "MassAI", "Enabled": true },
{ "Name": "StateTree", "Enabled": true }
```

按需追加：

```json
{ "Name": "ZoneGraph", "Enabled": true },
{ "Name": "SmartObjects", "Enabled": true },
{ "Name": "MassCrowd", "Enabled": true }
```

重新生成项目文件并编译 Editor。首次启用会显著增加编译模块数量，属正常现象。

## 项目 C++ 模块依赖（若写自定义 Processor）

```csharp
// Build.cs 示例
PrivateDependencyModuleNames.AddRange(new string[] {
    "MassEntity",
    "MassCommon",
    "MassSpawner",
    "MassSimulation",
    "MassAIBehavior",
    // 按需：MassMovement, MassLOD, MassRepresentation
});
```

## 编辑器中常见资产类型

| 资产 | 用途 |
|------|------|
| Mass Entity Config | 组合 Traits，定义 Agent 类型 |
| StateTree（Schema = Mass Behavior） | Mass AI 行为树 |
| Zone Graph（若用 ZoneGraph 导航） | 车道/通道数据 |
| Smart Object Definition | 可交互点（坐、用、躲等） |

## 常见问题

| 现象 | 可能原因 |
|------|----------|
| 启用 MassAI 编译失败 | 依赖插件未启用；需完整重编译 |
| Spawner 无实体生成 | Config 未指定、Trait 校验失败、Spawner 未触发 |
| StateTree 无法选 Mass Schema | MassAI 插件未启用或资产类型不对 |
| 有 Entity 无画面 | 未配置 Representation Trait / ISM |

## 最小验证步骤

1. 新建空白实验关卡
2. 创建 Mass Entity Config，添加 Movement + StateTree Trait
3. 放置 `AMassSpawner`，指向 Config，Count ≥ 20
4. PIE，打开 Mass 调试（`MassAIDebug` / Gameplay 调试命令，实践时补充有效命令）
