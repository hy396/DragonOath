# 01. Mass 核心概念

## Mass 解决什么问题

传统 `AActor` 每个实例都有完整 UObject 开销、组件树、Tick、复制通道。当同屏需要 **几十到上千** 个「逻辑简单、行为相近」的实体时，CPU 与内存压力会快速上升。

**Mass Entity** 采用 ECS 风格的数据导向设计：

> 用轻量 **Entity** 承载身份，用 **Fragment** 存数据，用 **Processor** 批量处理同类逻辑，避免「每个怪物一个完整 Actor」。

**MassAI** 在 MassGameplay 之上提供 AI 能力：StateTree 行为、导航（NavMesh / ZoneGraph）、SmartObject 交互、AI 相关复制等。

## 心智模型

```text
  UMassEntityConfigAsset（或 Spawner 配置）
           |
    Traits 组装 Entity Template
    （Movement / StateTree / Representation / LOD ...）
           |
      AMassSpawner 批量生成 Entity
           |
    UMassEntitySubsystem → FMassEntityManager
           |
    每帧按 Phase 执行 Processors
    （Query 匹配 Fragment 组合 → 批量读写）
           |
    MassRepresentation 同步到 ISM/Niagara/Actor 表现（可选）
```

## 关键术语

| 术语 | 含义 | 类比 |
|------|------|------|
| **Entity** | 轻量 ID，本身几乎无逻辑 | 不是 AActor，更像「行索引」 |
| **Fragment** | 挂在 Entity 上的纯数据结构 | 类似 ECS Component |
| **Tag** | 零大小标记 Fragment，用于 Query 过滤 | 状态标记、分类 |
| **Processor** | 对满足 Query 的 Entity 批量执行逻辑 | 替代每 Actor Tick |
| **Trait** | 编辑器可配「功能包」，构建 Template 时注入 Fragment/Processor 依赖 | 模块化拼装 Agent |
| **Query** | 声明 Processor 需要哪些 Fragment/Tag | 决定处理哪些 Entity |
| **Template** | 某类 Agent 的默认 Fragment 集合 | 「怪物 archetype」 |
| **Spawner** | 按配置批量实例化 Mass Entity | 刷怪点 / 刷怪潮 |

## UMassEntitySubsystem

引擎运行时模块 `MassEntity` 提供世界子系统：

```cpp
// MassEntitySubsystem.h 注释摘要：
// 为每个 UWorld 托管默认 FMassEntityManager 实例；
// MassGameplay 及相关插件默认使用它。
```

游戏代码通常通过 `UMassEntitySubsystem::GetEntityManager()` 访问实体数据。

## Processor 执行模型（要点）

- Processor 按 **Processing Phase** 分组（如 PrePhysics、DuringPhysics、PostPhysics…）
- 可通过 `ExecuteBefore` / `ExecuteAfter` / `ExecuteInGroup` 控制顺序
- 设计目标是 **批量、缓存友好、可并行**（部分 Processor 支持多线程）
- 逻辑应写在 Processor 中批量处理，而不是在每个 Entity 上挂 Tick

## Trait 与 Config

`UMassEntityTraitBase` 子类（如 `UMassStateTreeTrait`）在构建 Template 时：

1. 向 Template 添加 Fragment
2. 声明 Processor 依赖
3. 做配置校验（`ValidateTemplate`）

编辑器里通过 **Mass Entity Config** 资产组合多个 Trait，形成可复用的 Agent 类型。

## MassGameplay vs MassAI

| 层 | 插件 | 典型能力 |
|----|------|----------|
| 基础 | MassEntity（引擎 Runtime） | EntityManager、Processor 框架 |
| 玩法 | MassGameplay | Spawner、Movement、LOD、Representation、Replication |
| AI | MassAI | StateTree 行为、导航、SmartObject、MassAIReplication |

MassAI **依赖** MassGameplay；学 MassAI 前需理解 MassGameplay 的 Spawner / Movement / Representation。

## 与 DragonOath 的关系

横版 2.5D 动作 RPG 的怪物分两类：

**适合后期考虑 Mass：**

- 同屏大量杂兵、刷怪潮
- 背景装饰性敌人（逻辑简单）
- 只需基础追击/巡逻，战斗判定可简化或由「代表 Actor」处理

**不适合用 Mass 替代（首期保留 Actor）：**

- Boss、精英怪（复杂阶段、部位、技能轴）
- 强依赖 GAS 伤害、Buff、打击感、精确碰撞的敌人
- 需要完整动画通知、Montage、复杂受击表现的主体

项目架构文档已明确：**M5 先用 `EnemyCharacter + ASC + StateTree`**，Mass 在 M6/M8 压力出现后再引入。
