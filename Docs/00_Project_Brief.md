# 00. Project Brief

## 项目名称

正式工程名：DragonOath

中文暂定名：龙契战纪

内部代号：ProjectDrake

## 项目定位

DragonOath 是一款原创 2.5D 横版动作 RPG。项目可以参考经典横版动作页游的节奏与系统结构，但公开发布时必须使用原创名称、原创角色、原创剧情、原创 UI、原创图标、原创音效和原创美术资产。

核心目标不是复制某个旧项目，而是复现以下体验：

- 横版动作闯关
- 职业技能连招
- 怪物刷图与副本推进
- 装备和角色成长
- Boss 战
- 清晰直接的页游式操作反馈
- 联机架构下的即时战斗体验

## 目标用户体验

玩家进入游戏后，应能快速理解角色、技能、血蓝条、关卡目标和战斗节奏。第一版体验重点是“打起来顺手”，而不是系统数量。

优先级从高到低：

1. 操作响应
2. 打击反馈
3. 网络同步结构正确
4. 技能表现清晰
5. 成长循环成立
6. UI 信息明确
7. 内容量扩展

## 平台和视角

第一阶段目标平台：Windows PC

推荐表现方式：

- 2.5D 横版
- 固定或轻微跟随侧视摄像机
- 角色主要沿 X 轴移动
- Z 轴用于跳跃、击飞、浮空
- Y 轴纵深移动暂不作为第一阶段重点

## 首个可玩版本范围

首个可玩版本只追求一个完整闭环：

```text
启动游戏 -> 进入测试关卡 -> 操作角色 -> 使用普通攻击和技能 -> 击败怪物 -> 击败 Boss -> 获得掉落 -> 结算
```

必须包含：

- 一个玩家角色
- 一个职业原型
- 普通攻击三段
- 三个主动技能
- 基础 HP / MP
- 基础攻击、防御、暴击
- 三种普通怪
- 一个 Boss
- 一个测试副本
- 基础技能栏 UI
- 基础伤害数字
- 基础掉落和结算
- PIE Listen Server + Client 测试通过

暂不包含：

- 多职业
- 大量剧情
- 商城
- 充值
- 公会
- 排行榜
- 完整公网匹配服务
- 大规模多人同屏
- 宠物和坐骑
- 完整装备强化体系

## 版权和命名边界

开发时可以把经典横版动作页游作为体验参考，但不能直接复用：

- 原作名称
- 原作角色
- 原作职业名
- 原作剧情
- 原作地图
- 原作怪物
- 原作 UI 布局细节
- 原作图标
- 原作音效
- 原作素材

公开发布时使用 DragonOath / 龙契战纪方向，保持原创表达。

## 第一阶段开发原则

- 核心代码从第一天按联机架构编写。
- 不做完整公网服务，但要支持本地 Listen Server + Client 验证。
- GAS 是战斗、属性、技能、Buff/Debuff 的核心。
- Common UI 可用于正式 UI 框架，简单调试 UI 可先用普通 UMG。
- 系统先小闭环，后扩展。
- 美术先用占位资产验证手感，再替换正式资源。

## 成功标准

第一阶段成功不是“系统多”，而是以下事情成立：

- 玩家能稳定进入关卡。
- 普攻和技能能在客户端即时响应。
- 服务器权威处理伤害和死亡。
- 怪物和 Boss 的基本行为可同步。
- UI 能正确反映 HP、MP、冷却和结算。
- 关卡能从开始走到通关。
- 后续新增技能、怪物、装备不会推翻基础结构。
