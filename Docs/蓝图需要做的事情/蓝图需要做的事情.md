# 蓝图侧待办清单（第二阶段：进阶属性 + AttributeSet 拆分）

本清单对应 `Docs/06_Combat_Attribute_Design.md` 第二阶段落地后，**无法用 C++ 自动生成、必须在 UE 编辑器里手动完成**的工作。

代码侧已完成：`DOPlaySet` 已拆分为 `DOResourceSet` + `DOCombatSet`，`DOHealthSet` 新增 `HealthRegen` 与 `Healing` Meta，伤害结算已支持暴击/命中/闪避，并新增统一伤害入口 `ApplyDamageToTarget` 与 `GetAttackSpeed()`。

---

## 1. 配置伤害 GameplayEffect 资产（如 `GE_Damage`）

属性集已从 `DOPlaySet` 拆分，伤害 GE 的结算逻辑由 `UDOExecutionCalculation_Damage` 负责。请在伤害 GE 上配置：

- **Modifiers → Execution Calculation Class**：设为 `DOExecutionCalculation_Damage`（不要再用普通 Modifier 直接扣血）。
- **SetByCaller 数据**（由 `ApplyDamageToTarget` 通过代码传入，蓝图侧只需确保 GE 能接收这两个 Tag）：
  - `Data.Damage` —— 技能基础伤害值
  - `Data.DamageMultiplier` —— 技能伤害倍率
- **Source Tags（伤害来源标记）**：按施法者类型在 GE 的 Source Tags 上配置其一，缺省不配置则走"怪物/宠物"的命中判定路径：
  - `Damage.Type.Player` —— 玩家伤害，**跳过命中判定（必中）**
  - `Damage.Type.Monster` —— 怪物伤害，走命中/闪避判定
  - `Damage.Type.Pet` —— 召唤物/宠物伤害，走命中/闪避判定
- **吸血技能**：在 Source Tags 额外加 `Damage.CanLifeSteal`，`ApplyDamageToTarget` 会在服务端为来源按 `LifeStealRate * (SkillBaseDamage + AttackPower)` 回血。

> 说明：命中/闪避/暴击/减免公式全部在 ExecCalc 内完成，蓝图无需写运算，只需保证上面 Tag 配对。

## 2. 创建回复 GameplayEffect 资产

回复属性 `HealthRegen`（`DOHealthSet`）、`ManaRegen`（`DOResourceSet`）已就绪，但回复行为需两张蓝图 GE 驱动：

### `GE_Regen_Health`
- Duration Policy：`Infinite`
- Period：`1.0`
- Modifier：
  - Attribute：`Health`
  - Modifier Op：`Add`
  - Magnitude：**Attribute Based**，捕获源为自身的 `HealthRegen`（从 `UDOHealthSet` 选 `HealthRegen`）
- 备注：`HealthRegen` 为 `COND_OwnerOnly` 复制，Periodic GE 在拥有者（权威/本地）上计算，回血通过 `Health` 属性同步，无需客户端读取 Regen 值。

### `GE_Regen_Mana`
- Duration Policy：`Infinite`
- Period：`1.0`
- Modifier：
  - Attribute：`Mana`（来自 `UDOResourceSet`）
  - Modifier Op：`Add`
  - Magnitude：**Attribute Based**，捕获源为自身的 `ManaRegen`（从 `UDOResourceSet` 选 `ManaRegen`）

激活方式：角色出生时（或出生 GE 内）施加这两张 Infinite GE，并在死亡/离场时移除。

## 3. 伤害技能蓝图改用统一入口 `ApplyDamageToTarget`

新增的 C++ 函数（在 `UDOAbilitySystemComponent` 上）：

```text
ApplyDamageToTarget(
    DamageGEClass        : 伤害 GE 类（如 GE_Damage）
    Target               : 目标 Actor
    SkillBaseDamage      : 技能基础伤害（对应 Data.Damage）
    SkillDamageMultiplier: 技能伤害倍率（对应 Data.DamageMultiplier）
    SourceTags           : 伤害来源 Tag 容器（Damage.Type.*，吸血再加 Damage.CanLifeSteal）
    Level                : 等级，默认 1.0
)
```

请让原本"手动 `MakeOutgoingGameplayEffectSpec` + `ApplyGameplayEffectSpecToTarget`"的伤害技能，改为调用此函数。好处：自动处理 SetByCaller、并统一处理服务端吸血回血。

## 4. 迁移蓝图里的 `GetPlaySet()` 调用

第二阶段已删除 `UDOPlaySet` 与 `GetPlaySet()`。请在内容浏览器/蓝图中全局排查并替换：

- `GetPlaySet()` → 拆为：
  - `GetResourceSet()`（`UDOResourceSet`）：`Mana` / `MaxMana` / `Stamina` / `MaxStamina` / `ManaRegen`
  - `GetCombatSet()`（`UDOCombatSet`）：`AttackPower` / `DefensePower` / `MoveSpeed` / `CriticalRating` / `CritDamageRate` / `HitRating` / `EvasionRating` / `AttackSpeed` / `LifeStealRate`
  - `GetHealthSet()`（`UDOHealthSet`，原本就有）：`Health` / `MaxHealth` / `HealthRegen`
- `GetPlayerState` → `GetResourceSet()` / `GetCombatSet()` 是 `ADOPlayerState` 上的函数；若要从 ASC 取属性集，可用 `GetAbilitySystemComponent` → `GetSet<UDOCombatSet>()` 等（C++ 已提供）。

排查建议：在蓝图里搜索 `PlaySet`，以及在 `DragonOath` 内容目录下搜索引用了旧属性名的变量/函数。

## 5. 攻击动画播放速率接入 `GetAttackSpeed()`

`UDOGameplayAbility::GetAttackSpeed()` 已从 `UDOCombatSet::AttackSpeed` 读取（缺省 1.0）。请让攻击类 GA 在 `PlayMontageAndWait`（或播放蒙太奇节点）时，用 `GetAttackSpeed()` 作为 **PlayRate**，使攻速属性影响出手速度。

---

## 6. 怪物蓝图配置 `CharacterLevel`

`ADOCharacter` 上有 `CharacterLevel`（EditDefaultsOnly，默认 1），用于伤害公式的等级缩放。请在各怪物/NPC 蓝图的 Class Defaults 中按数据表填写正确等级（玩家目前固定读 Avatar 的 CharacterLevel，Phase 3 后由 PlayerState 覆盖）。
