- UE 5.8 工程、Git、Lyra 基础设施插件（Setly / GameplayMessageRouter 等）接入。
- C++ 功能域目录结构：`AbilitySystem/{Core, Abilities, Attributes, Pipeline}`、`Characters`、`Player`。
- 基类：`UDOAbilitySystemComponent`、`UDOGameplayAbility`、`ADOCharacter`、`ADOPlayerState`、`ADOPlayerCharacter`、`ADOPlayerController`。
- 属性集：`UDOHealthSet`、`UDOPlaySet`（含 Health / Mana / AttackPower / DefensePower / Damage 等）。
- `DOGameplayTag` 集中声明，含 `Profession`（DragonFighter / Mage / Archer）命名空间。
- Enhanced Input + GameplayTag 输入框架；`DA_InputConfig`、`IMC_Default`。
- **技能目录已按功能分类**：`Abilities/Core`（基类 + 配置资产）、`Abilities/Common`（通用技能）。`Combat/`、`Buff/`、`Passive/` 已约定、待首个技能时建目录。

## 2. 当前缺口（Gap 分析）

按"缺哪一层"整理，便于排优先级：

### 2.1 基础闭环缺口（M2 收尾）

- **属性 → HUD 同步**：属性集已建，但"属性变化绑定到 HUD、通过本地消息通知"尚未完成（Roadmap M2 任务）。
- **伤害 ExecCalc 落地**：`UDOExecutionCalculation_Damage` 框架在，但命中/暴击/格挡/减免/倍率完整链路与 `EffectContext` 写回需联调。
- **Common 技能尚无实际 C++ / 蓝图**：`DOGameplayAbility_NormalAttack`、`_Dash`、`_DashAttack` 已存在文件，但普攻三段、闪避（`Dodge`）等尚未按方案实现；冲刺/冲刺攻击仍是设计态。

### 2.2 技能体系缺口

- **职业技能资产缺失**：还没有任何一个职业的 `UDOAbilitySet` 资产，也没有 `UDOProfessionAbilityConfig` 资产；玩家登录时"按职业授予技能"链路未接通。
- **被动激活链路未闭环**：0→1 自动激活（`OnSpawn`）已规划，但 `SetDOAbilityLevel` 中 0↔1 副作用、`1→0` 清理（`CancelAbilityById` / GE 型被动映射）尚未实现与测试。
- **Combat / Buff / Passive 分类为空**：职业专属战技、辅助、被动目前只有命名空间，无实际技能类。

### 2.3 战斗与内容缺口（M3 起）

- 普攻三段连招、命中检测、TargetData 发送、服务器验证命中、伤害数字表现。
- 怪物 `ADOCharacter` + Enemy ASC + 受击/死亡；Boss 与副本流程尚未开始。
- 技能点经济（`SkillPoints` / `UDOSkillSet`）、`SkillTreeComponent` 前置/消耗管理尚未实现（升级方案已占位）。

## 3. 后续开发方向

按"先收口基础、再做闭环、后扩内容"三步走：

### 方向 A：M2 收尾 —— 把已有方案落地 + 属性 HUD 化

目标：让 GAS 数据真正在客户端可见、可联机验证。

- 实现 `冲刺与冲刺攻击方案` 的 C++ 与 GE 资产，打通双击 → 冲刺 → 冲刺攻击窗口。
- 联调 `伤害结算写回EffectContext`：ExecCalc 写回暴击/格挡，吸血改真实伤害，GameplayCue 能读 `IsCriticalHit`。
- 属性变化 → HUD 绑定（监听 `Message.UI.*` 或直接属性回调），满足 M2 验收。

### 方向 B：M3 基础战斗闭环 —— 普攻与命中

目标：一个能打的近战攻击最小闭环（联机权威）。

- 普攻三段 `NormalAttack`（放 `Common/`），含连段状态机、命中检测、TargetData、服务器验证。
- 伤害 GE + ExecCalc + 伤害数字飘字（复用方向 A 的 Context 写回）。
- 怪物受击与死亡（临时占位怪即可），验证"服务器决定伤害、客户端表现"。

### 方向 C：技能体系成型 —— 配置 + 授予 + 被动

目标：职业技能从"设计"变"可授予、可升级、可生效"。

- 建龙斗士 `UDOAbilitySet` 资产（NormalAttack=1、Dodge=1、FlameSlash=0、DragonBloodPassive=0、DeathAbility=事件）。
- 接通 `UDOProfessionAbilityConfig` + 玩家登录服务端 `GiveDOAbilitySet`。
- 实现 `SetDOAbilityLevel` 的 0↔1 被动激活/清理，验证升级方案骨架。
- 按"是否绑定职业"将技能归入 `Common/`（通用）与 `Combat/`（职业专属）；`Buff/`、`Passive/` 随首个技能建目录。

### 方向 D：M4–M5 内容扩展

- `Combat/`：龙斗士战技（如 `FlameSlash` 突进斩）、大招；后续法师/弓箭手专属技能各自建 `Combat/<Profession>/`。
- `Buff/`：增益/治疗/护盾类（pose + 特效 + GE，无弹道）。
- `Passive/`：龙血被动等属性/监听型被动。
- 怪物技能（建议另开 `AI/Abilities/` 或 `Abilities/Enemy/`，本次未建）：复用 `UDOAbilitySet` 结构，走 `UDOMonsterAbilityConfig`。

## 4. 后续内容清单（按 Abilities 分类）

对应已确定的目录结构，列出每类"接下来要做什么"：

### `Abilities/Core/`（已完成，后续只扩展）

- `DOGameplayAbility`：按需补充通用钩子（命中通知、连段工具、AbilityTask 封装）。
- `DOAbilitySet` / `DOProfessionAbilityConfig`：补编辑器数据校验（见职业技能配置方案"数据校验"节）。

### `Abilities/Common/`（通用基础动作，全职业共享）

| 技能 | 状态 | 下一步 |
|---|---|---|
| `NormalAttack` 普攻 | 文件在，待实现三段 | 连段状态机 + 命中 + TargetData + 伤害 |
| `Dash` 冲刺 | 设计完成（`冲刺与冲刺攻击方案`） | C++ 实现 + GE 资产 + 双击检测接入 |
| `DashAttack` 冲刺攻击 | 设计完成 | 同冲刺方案第三步 |
| `Dodge` 闪避（规划） | 仅有命名 | 与冲刺是否合并为"带无敌帧闪避"需确认（见下方备注） |

> 备注：现有 `Dash` 已带无敌帧，语义接近"闪避"。是否将 `Dodge`（职业配置示例里的 `Ability.DragonFighter.Dodge`）与 `Dash` 统一为同一技能，建议在实现前拍板，避免重复。

### `Abilities/Combat/`（职业专属主动战斗，规划中）

- 龙斗士：`FlameSlash`（战技）、大招类。
- 法师 / 弓箭手：各自专属战技与大招（M4 后）。
- 判定标准：该技能是某职业专属、有独特机制 → 放 `Combat/`；若后续职业多，进一步按 `Combat/<Profession>/` 细分。

### `Abilities/Buff/`（辅助，规划中）

- 增益 / 治疗 / 护盾：通常摆 pose + 特效 + GE，无弹道/命中逻辑。
- 例：攻击力 buff、群体治疗、护盾技能。

### `Abilities/Passive/`（被动，规划中）

- 简单属性型：直接 GE（最大生命 +%、攻击 +）。
- 复杂监听型：GA + `OnGranted`→`OnSpawn`（受击反击、低血护盾、暴击刷 CD、死亡复活）。

### 怪物技能（本次未建目录）

- 复用 `FDOAbilityGrant` / `UDOAbilitySet`，配置走 `UDOMonsterAbilityConfig`，由怪物自身 ASC 授予。

## 5. 推荐执行顺序（下一步清单）

按"低风险先落地、尽早联机验证"排序：

1. **属性 HUD 绑定**（M2 收尾）：先把现有属性显示出来，确立"属性变化 → 本地消息 → UI"范式。
2. **冲刺与冲刺攻击 C++ + GE**（已有方案）：打通第一个完整技能闭环，验证 LocalPredicted 位移/无敌/窗口。
3. **伤害 ExecCalc + EffectContext 联调**（执行中方案）：暴击/格挡写回、吸血改真实伤害、伤害数字。
4. **普攻三段 `NormalAttack`**（M3）：含命中检测 + TargetData + 服务器验证。
5. **职业技能资产与授予**：龙斗士 `UDOAbilitySet` + `UDOProfessionAbilityConfig`，玩家登录服务端授予。
6. **被动激活链路**：`SetDOAbilityLevel` 0↔1 副作用 + `1→0` 清理，接 `OnSpawn`。
7. **技能升级骨架**：`Server_RequestAbilityLevel` RPC + 校验 + 广播（技能点/树为依赖项，可先占位）。
8. **`Combat/` 首个职业战技**：如 `FlameSlash`，验证职业专属技能分类与配置。
9. **怪物与 Boss（M5）**：`ADOCharacter` + Enemy ASC + 受击/死亡 + 简单 AI。

每一步均保持 **PIE Listen Server + 1 Client** 自测，符合项目联机架构约定。
