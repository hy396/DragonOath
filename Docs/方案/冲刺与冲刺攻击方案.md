# 冲刺与冲刺攻击方案

## 原作机制

龙斗士原作中：

- 双击 A 或 D（移动方向键）触发方向冲刺
- 冲刺期间角色快速位移，带有短暂无敌帧
- 冲刺结束后有一段窗口时间，此时按下普攻键触发冲刺攻击
- 冲刺攻击与普通攻击是不同的技能（不同动画、不同伤害）

## 整体流程

```text
玩家双击 A/D
  -> 双击检测（角色层）
  -> 触发冲刺技能 GA_Dash
  -> 冲刺过程：位移 + 无敌 + 消耗体力
  -> 冲刺结束：施加 Status.DashAttackWindow 标签（持续 0.4s）
  -> 玩家按下普攻
  -> ASC 尝试激活所有绑定 InputTag.Ability.Primary 的技能
  -> GA_DashAttack 检测到 DashAttackWindow，激活成功
  -> GA_NormalAttack 检测到 DashAttackWindow，CanActivate 返回 false
  -> 只有冲刺攻击激活
```

## 双击检测

### 方案选择

Enhanced Input 自带双击检测 Modifier，但不够灵活（无法区分左右方向、无法自定义窗口时长）。

采用在角色层手动检测双击的方案。

### 实现位置

在 `ADOPlayerCharacter` 中新增双击检测逻辑。需要绑定 Move Action 的 `Started` 事件：

```cpp
// DOPlayerCharacter.h 新增
protected:
    void Input_MoveStarted(const FInputActionValue& InputActionValue);
    void TryStartDash();  // 触发冲刺技能

    // 双击检测（方向用于判断"同方向双击"，不传递给技能）
    float LastMovePressTime = -1.0f;
    float LastMovePressDirection = 0.0f;  // -1 = 左(A), 1 = 右(D)
    static constexpr float DoubleTapThreshold = 0.25f;  // 双击间隔阈值（秒）
```

### 绑定方式

在 `SetupPlayerInputComponent` 中增加 `Started` 事件的绑定：

```cpp
// 已有 Triggered 绑定（持续移动）
LyraIC->BindNativeAction(InputConfig, LyraGameplayTags::InputTag_Move, ETriggerEvent::Triggered, this, &ThisClass::Input_Move, /*bLogIfNotFound=*/ false);

// 新增 Started 绑定（检测双击）
LyraIC->BindNativeAction(InputConfig, LyraGameplayTags::InputTag_Move, ETriggerEvent::Started, this, &ThisClass::Input_MoveStarted, /*bLogIfNotFound=*/ false);
```

### 检测逻辑

```cpp
void ADOPlayerCharacter::Input_MoveStarted(const FInputActionValue& InputActionValue)
{
    const FVector2D Value = InputActionValue.Get<FVector2D>();

    // 只检测横向输入（A/D），忽略纯纵向（W/S）
    if (FMath::Abs(Value.X) < KINDA_SMALL_NUMBER)
    {
        return;
    }

    const float CurrentDirection = FMath::Sign(Value.X);
    const float CurrentTime = GetWorld()->GetTimeSeconds();

    // 同方向连续两次按下，且间隔在阈值内
    if (CurrentDirection == LastMovePressDirection
        && LastMovePressTime > 0.0f
        && (CurrentTime - LastMovePressTime) <= DoubleTapThreshold)
    {
        // 触发冲刺（方向由技能内部通过角色朝向获取）
        TryStartDash();

        // 重置检测状态，避免三连击重复触发
        LastMovePressTime = -1.0f;
        LastMovePressDirection = 0.0f;
    }
    else
    {
        LastMovePressTime = CurrentTime;
        LastMovePressDirection = CurrentDirection;
    }
}
```

### 注意事项

- `Started` 事件只在按键首次按下时触发一次，不会在持续按住时重复触发
- 双击检测只看方向键的 `Started`，不影响 `Triggered` 中的持续移动逻辑
- 如果玩家先按 A 再按 D，方向不同，不会触发冲刺（符合原作：只能同方向双击冲刺）
- 检测到双击后重置状态，避免持续连点重复触发

## 冲刺触发

### 不走普通输入标签路径

双击检测通过后，直接在角色层调用 ASC 激活冲刺技能，不走 `AbilityInputTagPressed` 流程。

原因：

- 双击是复合输入，不是单一的 InputAction
- 避免和 `InputTag.Ability.Dodge`（闪避键）混淆

### 触发代码

角色只负责检测双击和触发技能，状态检查和方向获取都由技能内部处理：

```cpp
void ADOPlayerCharacter::TryStartDash()
{
    UDOAbilitySystemComponent* DOASC = GetDOAbilitySystemComponent();
    if (!DOASC)
    {
        return;
    }

    // 通过 Tag 查找冲刺技能并激活
    FGameplayTagContainer DashTags;
    DashTags.AddTag(DragonOathGameplayTags::Ability::Movement::Dash);

    DOASC->TryActivateAbilitiesByTag(DashTags);
}
```

**职责分离**：

- **角色层**：双击检测 → 触发技能激活
- **技能层**：
  - `CanActivateAbility` 检查状态（冲刺中不能再冲刺、体力是否足够）
  - `GetActorForwardVector()` 获取冲刺方向（双击 A/D 时角色已转向）

不需要在角色层做状态预检查，也不需要传递方向参数。GAS 的 `CanActivateAbility` 会自动拦截不满足条件的激活请求。

### 后续优化：用 SpecHandle 直接激活

如果项目有 `AbilityId -> SpecHandle` 映射（见职业技能配置方案），可以用 AbilityId 直接激活：

```cpp
const FGameplayAbilitySpecHandle* Handle = DOASC->AbilityIdToSpecHandle.Find(DashAbilityId);
if (Handle)
{
    DOASC->TryActivateAbility(*Handle);
}
```

比 `TryActivateAbilitiesByTag` 更精确，避免遍历所有匹配 Tag 的技能。

### 对旧实现的改进分析

之前 HuanYu 项目中的 `HY_Dash` 用了 `ApplyRootMotionConstantForce`，整体思路正确，但有以下问题需要改进：

| 旧代码问题 | 改进方式 |
|---|---|
| `TriggerEventData->TargetData.Get(0)->GetHitResult()->ImpactNormal` 取方向，TargetData 为空时崩溃 | 双击检测时角色已经转向，直接用 `GetActorForwardVector()` 取方向，不传 TargetData |
| `TriggerEventData` 没有 null 检查就访问 `TargetData` | 不再依赖 EventData 传方向，消除空指针风险 |
| `K2_CommitAbility()` / `K2_EndAbility()` 蓝图风格函数 | 改用 C++ 原生 `CommitAbility` / `EndAbility` |
| Montage 四个回调都绑 `K2_EndAbility`，RootMotion Task 的 `OnFinish` 也绑，导致重复 EndAbility | 只用 RootMotion Task 的 `OnFinish` 结束技能，Montage 用单独回调处理取消 |
| `ActivationOwnedTags` 在构造函数硬编码标签 | 用 GE 施加 `Status.Dashing`，结束时自动移除 |
| 无 EndAbility 清理逻辑，标签和效果不会随技能取消而移除 | 在 `EndAbility` 中统一清理，区分正常结束和被取消 |
| `HasAuthorityOrPredictionKey` 包裹全部逻辑 | `LocalPredicted` 技能客户端和服务端都应该执行 ActivateAbility，不需要包裹 |

### 为什么用 ApplyRootMotionConstantForce

`LaunchCharacter` 虽然简单，但有几个缺点：

- 它是一次性脉冲速度，冲刺的加速/减速曲线不好控制
- 结束后角色会保持惯性滑行，不符合横版动作游戏的急停手感
- 与动画根位移的配合不如 RootMotion Source 自然

`UAbilityTask_ApplyRootMotionConstantForce` 的优势：

- 通过 RootMotion Source 驱动 CharacterMovementComponent，网络预测和回滚由引擎处理
- `IsAdditive = false` 可以覆盖现有速度，冲刺时不受移动输入干扰
- `ERootMotionFinishVelocityMode::ClampVelocity` 可以精确控制冲刺结束后的残留速度（设为 0 就是急停）
- 技能结束或被取消时，RootMotion Source 自动从 MovementComponent 移除，不会残留
- 支持 `StrengthOverTime` 曲线，可以做冲刺前段加速后段减速的手感

### 激活流程

```text
1. CanActivateAbility 检查：
   - 体力是否足够
   - 是否已经在冲刺中（Status.Dashing）
   - 是否被硬直/死亡等状态阻塞

2. ActivateAbility：
   a. CommitAbility（扣体力、触发冷却）
   b. 施加 Status.Dashing 标签（GE_DashStatus）
   c. 如果 bInvincibleDuringDash，施加无敌 GE（GE_DashInvincible）
   d. 播放冲刺蒙太奇（不阻塞，动画只做表现）
   e. 创建 ApplyRootMotionConstantForce Task 驱动位移
   f. Task 的 OnFinish 回调 -> 结束技能

3. 冲刺结束（Task 完成 / 被取消 / 被打断）：
   a. EndAbility 中统一清理
   b. 如果正常结束（非取消）：施加 Status.DashAttackWindow 标签
   c. 如果被取消：不施加窗口
```

### 实现文件

```cpp
// DOGameplayAbility_Dash.cpp

#include "AbilitySystem/Abilities/DOGameplayAbility_Dash.h"

#include "AbilitySystemComponent.h"
#include "Abilities/Tasks/AbilityTask_ApplyRootMotionConstantForce.h"
#include "Abilities/Tasks/AbilityTask_PlayMontageAndWait.h"
#include "AbilitySystem/Core/DOGameplayTag.h"
#include "GameFramework/Character.h"
#include "GameFramework/CharacterMovementComponent.h"

#include UE_INLINE_GENERATED_CPP_BY_NAME(DOGameplayAbility_Dash)

void UDOGameplayAbility_Dash::ActivateAbility(const FGameplayAbilitySpecHandle Handle, const FGameplayAbilityActorInfo* ActorInfo, const FGameplayAbilityActivationInfo ActivationInfo, const FGameplayEventData* TriggerEventData)
{
    Super::ActivateAbility(Handle, ActorInfo, ActivationInfo, TriggerEventData);

    if (!CommitAbility(Handle, ActorInfo, ActivationInfo))
    {
        EndAbility(Handle, ActorInfo, ActivationInfo, true, true);
        return;
    }

    UAbilitySystemComponent* ASC = GetAbilitySystemComponentFromActorInfo();
    ACharacter* Character = Cast<ACharacter>(GetAvatarActorFromActorInfo());
    if (!ASC || !Character)
    {
        EndAbility(Handle, ActorInfo, ActivationInfo, true, true);
        return;
    }

    // 施加冲刺状态标签（GE_DashStatus 包含 Status.Dashing）
    if (GE_DashStatus)
    {
        FGameplayEffectContextHandle Ctx = ASC->MakeEffectContext();
        FGameplayEffectSpecHandle Spec = ASC->MakeOutgoingSpec(GE_DashStatus, GetAbilityLevel(), Ctx);
        if (Spec.IsValid())
        {
            DashStatusHandle = ASC->ApplyGameplayEffectSpecToSelf(*Spec.Data.Get());
        }
    }

    // 无敌效果
    if (bInvincibleDuringDash && GE_DashInvincible)
    {
        FGameplayEffectContextHandle Ctx = ASC->MakeEffectContext();
        FGameplayEffectSpecHandle Spec = ASC->MakeOutgoingSpec(GE_DashInvincible, GetAbilityLevel(), Ctx);
        if (Spec.IsValid())
        {
            InvincibleHandle = ASC->ApplyGameplayEffectSpecToSelf(*Spec.Data.Get());
        }
    }

    // 播放冲刺蒙太奇。不等待蒙太奇结束，动画只负责表现，位移由 RootMotion Task 驱动。
    if (DashMontage)
    {
        UAbilityTask_PlayMontageAndWait* MontageTask = UAbilityTask_PlayMontageAndWait::CreatePlayMontageAndWaitProxy(
            this, NAME_None, DashMontage, 1.0f, NAME_None, false);
        // 蒙太奇被中断时取消技能（被击、死亡等）
        MontageTask->OnCancelled.AddDynamic(this, &ThisClass::K2_EndAbility);
        MontageTask->ReadyForActivation();
    }

    // 冲刺方向：角色当前朝向。双击 A/D 时角色已经转向对应方向。
    const FVector DashDirection = GetDashDirection(ActorInfo);

    // 创建 RootMotion 恒力 Task
    UAbilityTask_ApplyRootMotionConstantForce* RootMotionTask =
        UAbilityTask_ApplyRootMotionConstantForce::ApplyRootMotionConstantForce(
            this,
            NAME_Dash,                    // 实例名称
            DashDirection,                 // 世界空间方向（内部会归一化）
            DashStrength,                  // 力的大小
            DashDuration,                  // 持续时间
            false,                         // IsAdditive = false：覆盖现有速度，冲刺期间不受移动输入影响
            nullptr,                       // StrengthOverTime：null 表示恒力，后续可配曲线做加速减速
            ERootMotionFinishVelocityMode::ClampVelocity,  // 结束时限制残留速度
            FVector::ZeroVector,           // SetVelocityOnFinish（未使用）
            GetMaxSpeed(ActorInfo),        // ClampVelocityOnFinish：结束时速度限制为最大移动速度
            false                          // bEnableGravity：冲刺期间关闭重力
        );

    // RootMotion Task 完成时结束技能
    RootMotionTask->OnFinish.AddDynamic(this, &UDOGameplayAbility_Dash::OnDashFinished);
    RootMotionTask->ReadyForActivation();
}

void UDOGameplayAbility_Dash::OnDashFinished(FGameplayAbilitySpecHandle Handle, const FGameplayAbilityActorInfo* ActorInfo, const FGameplayAbilityActivationInfo ActivationInfo)
{
    // RootMotion Task 自然完成 = 冲刺正常结束
    EndAbility(Handle, ActorInfo, ActivationInfo, true, false);
}

void UDOGameplayAbility_Dash::EndAbility(const FGameplayAbilitySpecHandle Handle, const FGameplayAbilityActorInfo* ActorInfo, const FGameplayAbilityActivationInfo ActivationInfo, bool bReplicateEndAbility, bool bWasCancelled)
{
    UAbilitySystemComponent* ASC = GetAbilitySystemComponentFromActorInfo();
    if (ASC)
    {
        // 移除冲刺状态标签
        if (DashStatusHandle.IsValid())
        {
            ASC->RemoveActiveGameplayEffect(DashStatusHandle);
            DashStatusHandle.Reset();
        }

        // 移除无敌效果
        if (InvincibleHandle.IsValid())
        {
            ASC->RemoveActiveGameplayEffect(InvincibleHandle);
            InvincibleHandle.Reset();
        }

        // 只有正常结束才施加冲刺攻击窗口
        if (!bWasCancelled && GE_DashAttackWindow)
        {
            FGameplayEffectContextHandle Ctx = ASC->MakeEffectContext();
            FGameplayEffectSpecHandle Spec = ASC->MakeOutgoingSpec(GE_DashAttackWindow, GetAbilityLevel(), Ctx);
            if (Spec.IsValid())
            {
                Spec.Data.Get()->SetDuration(FScalableFloat(DashAttackWindowDuration), true);
                ASC->ApplyGameplayEffectSpecToSelf(*Spec.Data.Get());
            }
        }
    }

    Super::EndAbility(Handle, ActorInfo, ActivationInfo, bReplicateEndAbility, bWasCancelled);
}

FVector UDOGameplayAbility_Dash::GetDashDirection(const FGameplayAbilityActorInfo* ActorInfo) const
{
    // 横版游戏：冲刺方向 = 角色当前朝向。
    // 双击 A/D 时角色已经转向对应方向，直接取 ActorForwardVector。
    if (const AActor* Avatar = GetAvatarActorFromActorInfo())
    {
        return Avatar->GetActorForwardVector();
    }
    return FVector::ForwardVector;
}

float UDOGameplayAbility_Dash::GetMaxSpeed(const FGameplayAbilityActorInfo* ActorInfo) const
{
    if (const ACharacter* Character = Cast<ACharacter>(GetAvatarActorFromActorInfo()))
    {
        if (const UCharacterMovementComponent* MoveComp = Character->GetCharacterMovement())
        {
            return MoveComp->GetMaxSpeed();
        }
    }
    return 500.0f;
}
```

### 关键改进对比

**IsAdditive = false**：冲刺期间覆盖角色现有速度。玩家冲刺时即使还在按方向键，也不会影响冲刺轨迹。旧代码也用了 `false`，这是正确的。

**ERootMotionFinishVelocityMode::ClampVelocity**：冲刺结束后不是直接归零，而是限制为 `GetMaxSpeed()`。这样冲刺结束瞬间角色能平滑过渡到正常移动速度，不会突然卡住。如果想做急停手感，改为 `SetVelocity` 并设 `FVector::ZeroVector`。

**bEnableGravity = false**：冲刺期间关闭重力，避免角色在斜坡或台阶上冲刺时被重力拉偏。如果冲刺需要贴地，可以改为 `true`。

**Montage 不阻塞技能结束**：蒙太奇只做视觉表现，技能结束完全由 RootMotion Task 的 `OnFinish` 控制。这样即使动画比冲刺时间长（比如冲刺 0.3s 但动画 0.5s），技能也会在位移结束时正确进入冲刺攻击窗口。旧代码四个蒙太奇回调都绑 `K2_EndAbility`，会导致动画一结束就 EndAbility，但 RootMotion 可能还在跑。

**EndAbility 统一清理**：所有状态标签和 GE 的移除都放在 `EndAbility` 中，无论是正常结束还是被取消都会执行。`bWasCancelled` 区分是否施加冲刺攻击窗口。旧代码没有这个逻辑，被击或死亡时冲刺标签不会移除。

### 体力消耗

冲刺消耗 Stamina。Cost GE 配置：

```text
GE_DashCost
  Modifier: Stamina, Operation: Add, Value: -25
  Duration: Instant
```

GAS 的 `CommitAbility` 会自动检查 Cost GE。如果 Stamina 不足，`CommitAbility` 返回 false，冲刺不激活。

### 后续可扩展：StrengthOverTime 曲线

如果需要冲刺前段加速后段减速的手感，可以配置一条 `UCurveFloat`：

```cpp
// 曲线：(0.0, 1.0) -> (0.5, 1.0) -> (1.0, 0.3)
// 前 50% 时间全力，后 50% 衰减到 30%
UPROPERTY(EditDefaultsOnly, BlueprintReadOnly, Category = "DO|Dash")
TObjectPtr<UCurveFloat> StrengthOverTime;
```

然后传给 `ApplyRootMotionConstantForce` 的 `StrengthOverTime` 参数。曲线的 X 轴范围是 `[0.0, 1.0]`，0 是冲刺开始，1 是冲刺结束。

## 冲刺攻击技能 UDOGameplayAbility_DashAttack

### 基本信息


### 核心设计

冲刺攻击和普通攻击共用 `InputTag.Ability.Primary` 输入标签。当玩家按下普攻键时，ASC 会尝试激活所有绑定该输入标签的技能。通过 `CanActivateAbility` 中的标签检查来决定哪个技能实际激活：

```cpp
bool UDOGameplayAbility_DashAttack::CanActivateAbility(const FGameplayAbilitySpecHandle Handle, const FGameplayAbilityActorInfo* ActorInfo, const FGameplayTagContainer* SourceTags, const FGameplayTagContainer* TargetTags, FGameplayTagContainer* OptionalRelevantTags) const
{
    if (!Super::CanActivateAbility(Handle, ActorInfo, SourceTags, TargetTags, OptionalRelevantTags))
    {
        return false;
    }

    // 只有冲刺攻击窗口期间才能激活
    const UAbilitySystemComponent* ASC = ActorInfo->AbilitySystemComponent.Get();
    if (!ASC || !ASC->HasMatchingGameplayTag(DragonOathGameplayTags::Status::DashAttackWindow))
    {
        return false;
    }

    return true;
}
```

### 普攻的配合修改

`UDOGameplayAbility_NormalAttack` 需要在 `CanActivateAbility` 中检查：如果处于冲刺攻击窗口，返回 false，让冲刺攻击优先激活：

```cpp
bool UDOGameplayAbility_NormalAttack::CanActivateAbility(const FGameplayAbilitySpecHandle Handle, const FGameplayAbilityActorInfo* ActorInfo, const FGameplayTagContainer* SourceTags, const FGameplayTagContainer* TargetTags, FGameplayTagContainer* OptionalRelevantTags) const
{
    if (!Super::CanActivateAbility(Handle, ActorInfo, SourceTags, TargetTags, OptionalRelevantTags))
    {
        return false;
    }

    const UAbilitySystemComponent* ASC = ActorInfo->AbilitySystemComponent.Get();

    // 冲刺攻击窗口期间，普攻让位给冲刺攻击
    if (ASC && ASC->HasMatchingGameplayTag(DragonOathGameplayTags::Status::DashAttackWindow))
    {
        return false;
    }

    return true;
}
```

### 冲刺攻击激活流程

```text
1. CanActivateAbility 检查通过（DashAttackWindow 激活）
2. ActivateAbility：
   a. CommitAbility
   b. 移除 Status.DashAttackWindow（一次性消耗，避免重复触发）
   c. 播放冲刺攻击动画
   d. 等待动画命中通知（Anim Notify -> AbilityTask_WaitGameplayEvent）
   e. 命中时应用伤害 GE
   f. 动画结束后 EndAbility
```

### 冲刺攻击移除窗口标签

冲刺攻击激活后应该消耗掉冲刺攻击窗口，避免在窗口内连续触发：

```cpp
void UDOGameplayAbility_DashAttack::ActivateAbility(...)
{
    // ...

    // 消耗冲刺攻击窗口
    UAbilitySystemComponent* ASC = ActorInfo->AbilitySystemComponent.Get();
    if (ASC)
    {
        ASC->RemoveLooseGameplayTag(DragonOathGameplayTags::Status::DashAttackWindow);
    }

    // 播放动画、等待命中...
}
```

## 输入配置

冲刺不需要单独的 InputAction 或 InputMappingContext。双击检测基于已有的 Move InputAction 的 `Started` 事件，不需要新增按键绑定。

冲刺攻击也不需要单独的 InputAction，它共用 `InputTag.Ability.Primary`（普攻键）。

## 网络考虑

### 冲刺

- `LocalPredicted`：客户端预测冲刺位移，服务端最终确认
- `LaunchCharacter` 本身支持网络预测，客户端会立即执行，服务端会纠正
- `Status.Dashing` 标签通过 GE 复制，服务端施加后自动同步到客户端
- 无敌效果在服务端生效，客户端看到的是表现

### 冲刺攻击

- `LocalPredicted`：客户端预测激活，立即播放动画
- 伤害 GE 在服务端应用（通过 TargetData 验证流程）
- `Status.DashAttackWindow` 通过 GE 复制，所有客户端都能看到

### 双击检测

- 双击检测只在客户端本地执行（控制自己角色的客户端）
- 检测到双击后，客户端通过 `TryActivateAbility` 发起预测激活
- 服务端会收到激活请求并做最终判断（体力是否足够、是否在硬直中等）

## 边界情况处理

### 冲刺被中断

如果冲刺过程中角色被击中（虽然有无敌帧，但如果关闭了无敌）、死亡、进入硬直：

- `EndAbility` 会被调用（通过 `OnGameplayEffectApplied` 或 `CancelAbilities` 触发）
- `Status.Dashing` 标签需要随 GE 移除自动清除
- 不施加 `Status.DashAttackWindow`（冲刺被中断不算正常结束）

可以在 `OnEndAbility` 回调中检查是否是正常结束：

```cpp
void UDOGameplayAbility_Dash::EndAbility(const FGameplayAbilitySpecHandle Handle, const FGameplayAbilityActorInfo* ActorInfo, const FGameplayAbilityActivationInfo ActivationInfo, bool bReplicateEndAbility, bool bWasCancelled)
{
    if (!bWasCancelled)
    {
        // 正常结束：施加冲刺攻击窗口
        // ...
    }
    // 取消结束：不施加窗口

    Super::EndAbility(Handle, ActorInfo, ActivationInfo, bReplicateEndAbility, bWasCancelled);
}
```

### 体力不足

`CommitAbility` 会检查 Cost GE。体力不足时返回 false，冲刺不激活。可以播放一个"体力不足"的音效或提示。

### 冲刺冷却

可以给冲刺加一个短冷却（如 0.5s），防止双击检测抖动导致连续触发。配置在 `GA_Dash` 的 `CooldownGameplayEffectClass` 中。

### 冲刺攻击窗口超时

如果玩家在窗口内没有按普攻，`Status.DashAttackWindow` 的 GE 会自动过期，标签自动移除。此时按普攻恢复正常攻击。

### 同时按多个方向键

如果玩家同时按 A+D，`Input_MoveStarted` 中 `Value.X` 的绝对值可能很小或为零。加一个阈值判断，只有明确的方向才参与双击检测。
