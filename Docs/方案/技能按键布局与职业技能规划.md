# 技能按键布局与职业技能规划

> 阶段：基于当前进度（M1–M2 之间）的技能层规划，作为 `Abilities/` 分类落地与职业技能配置（`DOAbilitySet` / `UDOProfessionAbilityConfig`）的输入。
> 关联：`Docs/03_Milestone_Roadmap.md`、`Docs/ProfessionAbilityConfigDesign.md`、`Docs/方案/冲刺与冲刺攻击方案.md`、`Docs/方案/后续开发方向与内容.md`、`AGENTS.md` 的 "Abilities 分类约定"。
> 游戏形态：**2.5D 横版、纯键盘操作**。

### 1.1 需要新增的 InputTag

当前 `DOGameplayTag` 的 `InputTag` 命名空间已有 `Primary`、`Dodge` 等。需补充：

```text
InputTag.Ability.Skill1 .. Skill8
InputTag.Ability.Buff1  .. Buff4
InputTag.Move.Jump
```

（`Dodge` 概念与"双击 A/D 触发冲刺"合并为同一能力，不再单独占键；职业配置示例里的 `Ability.DragonFighter.Dodge` 统一指向双击触发的冲刺技能。）

### 1.2 键位 → AbilityId 映射规则

- 每个职业的 `UDOAbilitySet` 里，`FDOAbilityGrant` 的 `InputTag` 按上表写死键位，`AbilityId` 用 `Ability.<职业>.<技能名>` 形式（职业用 `Profession` Tag 区分，不用枚举）。
- 8 个技能键 + 4 个辅助键 = 每职业 **12 个可绑定技能位**；被动不占键（靠 `OnGranted`/`OnSpawn` 自动激活，或在 `AbilitySet` 中 `InitialLevel=0` 待升级）。
- 早期 MVP 不必填满 12 格：未实装技能配 `InitialLevel=0`（UI 显示但未学习，输入被 `CanActivateAbility` 拦截）。

## 5. 目录落位（对照 Abilities 分类约定）

```text

  Core/        技能基类 + 配置资产（已完成）
  Common/      普攻 NormalAttack、冲刺 Dash、冲刺攻击 DashAttack（双击触发，不占键）
  Combat/
    DragonFighter/  突进斩、浮空斩、龙焰斩、旋风斩、龙吼、龙血冲锋、招架、龙怒
    Mage/           火球术、冰锥、闪电链、陨石术、闪现、寒冰新星、奥术飞弹、元素风暴
    Archer/         精准射击、多重射击、穿透箭、箭雨、后撤步、陷阱、鹰眼、万箭齐发
  Buff/
    DragonFighter/  龙血狂暴、龙鳞护盾、嗜血、疾风战意
    Mage/           法力护盾、智慧祝福、冥想、屏障
    Archer/         疾风步、锐利箭头、治疗术、荆棘护甲
```

## 6. 2.5D 横版设计注意点

- **面向释放**：所有技能以角色 `GetActorForwardVector()`（双击/移动决定的朝向）为方向，不做八方向瞄准。
- **弹道水平**：远程技能（火球、冰锥、箭）沿水平轴飞行，配合 2.5D 摄像机。
- **AoE 范围**：以自身或前方矩形/扇形为主，不依赖 3D 空间判定。
- **闪避双击**：`A`/`D` 双击触发，方向由当前朝向决定，天然适配横版左右位移（见 `冲刺与冲刺攻击方案`）。
- **大招 `M`**：建议全屏或超大前方范围，演出强、CD 长。

## 7. 实现建议顺序

1. 在 `DOGameplayTag` 的 `InputTag` 命名空间新增 `Skill1..8`、`Buff1..4`、`Move.Jump`。
2. `DA_InputConfig` 中把上述 InputTag 绑定到对应 `InputAction`（H/L/U/I/O/P/N/M、5/6/7/8、K）。
3. 先做龙斗士一套：龙斗士 `UDOAbilitySet`（含 NormalAttack=1、Dash=双击、FlameSlash=0、DragonBloodPassive=0、DeathAbility=事件），接通 `GiveDOAbilitySet`。
4. 按 `Common/` → `Combat/DragonFighter/` → `Buff/DragonFighter/` → `Passive/` 顺序落地技能类与蓝图。
5. `M` 大招、`N` 招架等留到职业战斗手感稳定后再做。
6. 法师 / 弓箭手复用同一套键位与 `FDOAbilityGrant` 结构，仅替换 `AbilityClass` 与 `AbilityId`。

## 8. 待确认 / 备注

- **8 技能格 ≠ 一次填满**：早期 MVP 可只实装 `H/L/U` + 大招 `M`，其余配 `InitialLevel=0`。
- **闪避不占键**：双击 `A/D` 已写死，与职业配置示例里的 `Dodge` 合并为同一冲刺能力，注意 `UDOAbilitySet` 里 `Dodge` 的 `AbilityClass` 指向 `GA_Dash`。
- **被动激活链路**：`0→1` 自动激活（`OnSpawn`）与 `1→0` 清理尚未实现，需先完成 `ProfessionAbilityConfigDesign` 的被动链路再接通被动技能。
- **辅助键 `5/6/7/8` 在键盘右侧数字行**：若与聊天/快捷栏冲突，可改小键盘或 `F1..F4`，此处按用户给定方案写死。
