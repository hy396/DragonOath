# 通用学习版 · 新手引导系统

> 与引擎无关（Unity / Unreal / Cocos 同理）。重点讲"为什么这样拆 + 行业里真实长什么样"。
> 本版以一份较完整的行业资料为底稿，补足概念、分类、前后端分工，便于学习理解。

## 1. 它解决什么问题

新手玩家第一次进游戏，不知道该看哪里、点什么、做什么。引导系统把"教学目标"编排成一串**有序步骤**，每一步告诉玩家"看这里、做这个"，做完进下一步；中途退出也能**接着来**（断点续引）。它是降低流失率、提升上手体验的关键功能。

### 市面上三种实现路线
1. 公司封装好的闭源引导框架（中高级开发维护，最省事但不可控）
2. 第三方插件工具（开箱即用，但要适配项目（本文档不推荐，除非项目极小））
3. 项目自研手写框架（最灵活，本文档聚焦这种）

## 2. 企业级分类（先分大类，再分细类）

### 两大引导大类
| 大类 | 行为 | 适用 |
|---|---|---|
| **流程式引导（强制主线）** | 强制步骤、跟随点击、强制解锁，卡住流程，必须完成才能推进 | 新手期核心教学 |
| **气泡弹窗弱引导** | 仅弹窗提示，不卡主线 | 新系统解锁、活动提示、回流提醒 |

### 细分引导类型
- UI 点击引导（最常用）
- 3D 场景/角色引导（让镜头/角色走到某处）
- 剧情对话引导
- 玩法规则弹窗引导
- 红点 + 弹窗组合引导（引导入口用红点点亮，见红点系统）

## 3. 完整体系架构（行业级）

### 客户端核心职责
1. 渲染遮罩、高亮、箭头、提示文字
2. 监听步骤推进：下一步 / 跳过 / 重新触发
3. 监听目标控件：加载、显隐、挂载、销毁
4. 步骤条件判断：参数、状态、配置表校验
5. 流程分屏、多段动画、多渠道
6. 引导期间全局事件监听、跳转拦截（强制等待引导）

### 服务端三种模式（关键：引导要不要后端参与）
| 模式 | 后端参与程度 | 适用 |
|---|---|---|
| **A：纯客户端离线** | 后端完全不参与，步骤+进度全本地缓存 | 小游戏、超休闲、轻度手游 |
| **B：下发配置 + 同步进度（中大型通用）** | 后端下发步骤/文案/解锁条件/跳转 ID；每一步完成情况同步玩家库；断线重连读云端 | 推图主线、任务引导、分阶段开放 |
| **C：强服务驱动（重度 MMO/RPG）** | 按战力/副本进度/付费状态动态下发引导分支；双端控制解锁，客户端只渲染 | 重度 RPG、MMO |

> 选型的判断标准：游戏越重、越联机、越怕作弊，后端参与越多。DragonOath 是联机 RPG，应落在 **B / C**（见项目相关版）。

### 前后端详细分工（以 B/C 为例）
- **客户端做什么**：遮罩/高亮/箭头、状态机流转、屏幕坐标转换、弹窗监听、场景切换监听、引导标记/重置、多语言、维护引导配置表（步骤 ID/触发条件/关联 UI/文案/动画/前置）、存玩家进度（已完成步骤/是否跳过）、接收新配置刷新开关、灰度分服、埋点日志、操作拦截、配置表读取卡点、引导预览面板。
- **后端做什么**：下发全量配置+进度、返回已完成/未解锁/开服天数、状态机与进度同步、配置触发条件、接收步骤完成上报入库、全服进度永久持久。

### 引导条件类型（两端都要支持）
- **客户端本地条件**：点击指定按钮、打开指定弹窗、关卡通关、剧情点下一步、达到指定等级、通关副本、获得指定道具、开服第 N 天、完成新手任务、首次充值/交手……
- **服务端下发条件**：引导分支（新手/回流/老玩家走不同线）、优先级排队（新引导 > 红点 > 弹窗）、防冲突（多引导自动过滤低优先级）、防重复（结束历史不重触发）、埋点转化率分析、双端配置校验（不等则卡死，策划在编辑器改多端）。

## 4. 三个核心概念（学习用最小模型）

| 概念 | 说明 |
|---|---|
| **步骤 Step** | 最小单元：文案 + 高亮目标 + 完成条件 |
| **流程 Flow** | 步骤的有序集合，支持推进/跳过/完成 |
| **断点 Checkpoint** | 当前进行到的步骤，持久化后下次可续引 |

### 数据结构（伪代码，引擎无关）
```text
Step {
  id              : string        // 唯一标识
  title           : string        // 文案
  highlightTarget : string        // 高亮 UI 元素 id（如 "btn_skill"）
  completeWhen    : Event[]       // 收到这些事件之一即完成（如 "move_done"）
  skippable       : bool
}

Tutorial {
  id       : string
  steps    : Step[]                // 有序
  prereq   : string[]              // 前置引导（全完成才可开始）
}

SaveData {
  completedSteps : string[]        // 已完成
  currentStepId  : string          // 断点
  everStarted    : bool
  saveVersion    : int             // 配置大改时作废旧档
}
```

### 状态机视角
```text
[Idle] --start--> [Step_0] --done--> [Step_1] --done--> ... --done--> [Finished]
                 Step_i --skip--> Step_{i+1}   (仅当 skippable)
```
- `EnterStep`：展示文案 + 高亮 + 订阅完成事件 + **立即存盘断点**。
- 完成事件到达 → `CompleteStep`：记录存档 + 取消高亮 + 进入下一步。
- 全部完成 → `Finished`：清断点。

## 5. 断点续引（关键）
- 进入游戏先读 `SaveData.currentStepId`：有值 → 直接 `EnterStep(该步)`；无值 → 等触发条件再 `start`。
- 每次进入/完成都**立即存盘**，异常退出也能续。

## 6. 焦点高亮怎么做（通用技巧）
全屏半透明遮罩，中间"挖个洞"对准目标控件：
```text
画 4 个黑色半透明矩形：
  上： (0,0)            → (屏宽, 洞顶Y)
  下： (0,洞顶Y+洞高)   → (屏宽, 屏高)
  左： (0,洞顶Y)        → (洞左X, 洞顶Y+洞高)
  右： (洞左X+洞宽,洞顶Y)→ (屏宽, 洞顶Y+洞高)
洞边缘画一圈脉冲边框（用时间做呼吸动画）
```
- 目标屏幕坐标：取控件布局包围盒（屏幕绝对坐标）。
- 坐标来源可以是"控件引用"，也可以是直接传"屏幕坐标+尺寸"。

## 7. 配套测试需求（行业常见）
1. 快速切换引导步骤调试
2. 分段测试：只看当前、屏蔽无用信息
3. 自动屏蔽代码弹窗，不用手动遮挡
4. 统一日志格式，多人协作看得懂
5. 反馈 BUG 直接查日志定位代码位置
6. 记录操作流程+场景+时间，还原玩家路径
7. 统计高频警告、隐性 BUG（不闪退但逻辑异常）

## 8. 学习要点小结
- **引导只认"事件"，不认触发方**：完成条件用事件/消息表达，业务爱怎么触发都行。
- **UI 解耦**：逻辑（状态机/存档）与表现（遮罩/文案）分离，表现层只听逻辑层通知。
- **可数据驱动**：步骤配在表/资源里，策划改步骤不动代码。
- **要不要后端**：看游戏轻重（模式 A/B/C），后端负责配置下发 + 进度持久 + 防作弊。
