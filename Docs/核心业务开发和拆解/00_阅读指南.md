# 核心业务系统拆解 · 阅读指南

本目录提供**同一套系统（新手引导 / 红点 / 日志）**的两种拆解视角：

| 目录 | 适用场景 | 特点 |
|---|---|---|
| [项目相关版/](./项目相关版) | 在 DragonOath 里真正落地时 | 咬合本项目的 GAS、职业 Tag、消息总线、已有日志通道；修正了子系统归属；引用真实类名与文件 |
| [通用学习版/](./通用学习版) | 理解概念、跨项目复用思路 | 引擎无关；用通用设计模式（观察者 / 状态机 / 生产者消费者）讲清楚“为什么这样做” |

## 三套系统是什么（一句话）

- **新手引导**：把“教玩家玩游戏”拆成有序步骤，逐步推进、可断点续引、可高亮焦点控件。
- **红点**：用一棵树表达“哪里有未处理的事”，父节点由子节点自动 OR 出来，变化才通知 UI。
- **日志**：分级、线程安全落盘、崩溃抓栈，是所有系统的横切基础设施。

## 与项目架构的关键约定（项目相关版遵循）

- 引导 / 红点是**单个本地玩家**的状态 → 用 `ULocalPlayerSubsystem`（而非全局 `UGameInstanceSubsystem`），联机/多本地玩家不串号。
- 日志是**全局**关注 → 仍用 `UGameInstanceSubsystem`。
- UI 只监听 `GameplayMessageRouter` 的 `Message.UI.*` 刷新，不反向改规则（见 `Docs/04_Local_Message_Bus.md`）。
- 标识统一用 `GameplayTag`（不用枚举），已新增 `Message.UI.RedDot.Changed` / `Message.UI.Tutorial.*`。
- 日志复用已有的 `DOLogChannels`（`LogDragonOath*`），在其下游加持久化与堆栈捕获。
