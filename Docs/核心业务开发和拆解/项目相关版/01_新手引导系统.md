# 项目相关版 · 新手引导系统

> 落地于 DragonOath：咬合 GAS、职业 Tag、消息总线、SaveGame、UE 权威端（联机 RPG 走"模式 B/C"）。
> 子系统归属 `ULocalPlayerSubsystem`（每玩家本地进度）；权威端进度存 `ADOPlayerState`（服务端权威）。

## 0. 选型：DragonOath 走"模式 B/C"

朋友给的 Unity 资料把引导分 A/B/C 三模式。DragonOath 是**联机 RPG**，应落在：
- **模式 B（下发配置 + 同步进度）**：引导步骤配在 `DA_TutorialDefinition`（本地资产即"下发配置"），进度同步到 `ADOPlayerState` 并复制给客户端。
- **模式 C（强服务驱动）**：后续若需按战力/副本进度动态分支，由服务端（DS/Listen Server 的 C++ 权威逻辑）决定走哪条引导。

客户端只渲染、只存本地断点；**真值（是否已完成、当前步骤）以 `PlayerState` 复制为准**，防作弊、断线重连校准。

## 1. 模块划分（真实目录）
```text
Source/DragonOath/Systems/Tutorial/
  DOTutorialSubsystem.h/.cpp        ULocalPlayerSubsystem：本地流程调度、断点续引、监听消息
  DOTutorialTypes.h                 FDOTutorialStep / EDOTutorialHighlightShape
  DOTutorialDefinition.h/.cpp       UPrimaryDataAsset：一条引导的步骤配置（=模式B的"下发配置"）
  DOTutorialSaveGame.h/.cpp         USaveGame：本地断点缓存（slot = TutorialSave_<LocalPlayerId>）
  DOTutorialHighlightWidget.h/.cpp  UMG 全屏遮罩：挖孔高亮 + 脉冲边框
Source/DragonOath/Player/
  DOPlayerState                    // 权威端：存 CompletedTutorials / CurrentStep（复制给客户端）
```

## 2. 为什么 LocalPlayerSubsystem + PlayerState 双归属
- 引导是**单个玩家**的进度：本地表现/断点用 `ULocalPlayerSubsystem`（跨关卡不丢、联机不串号）。
- 权威真值用 `ADOPlayerState`：服务器决定"该玩家完成了哪些引导、当前步"，经属性复制下发，客户端 LocalPlayer 子系统照此渲染。
- 日志用 `UGameInstanceSubsystem`（全局合理）；红点同理用 `LocalPlayerSubsystem`（见 02），其权威端也在 `PlayerState`。

## 3. 数据结构（贴项目命名）
```cpp
USTRUCT(BlueprintType)
struct FDOTutorialStep
{
    GENERATED_BODY()
    UPROPERTY(EditAnywhere) FGameplayTag StepId;          // 如 "Tutorial.Newbie.Move"
    UPROPERTY(EditAnywhere) FText Title, Description;
    UPROPERTY(EditAnywhere) FGameplayTag FocusWidgetTag;  // 高亮控件标识
    UPROPERTY(EditAnywhere) EDOTutorialHighlightShape HighlightShape = Rectangle;
    UPROPERTY(EditAnywhere) TArray<FGameplayTag> CompleteOnTags; // 监听到其一即完成（GameplayMessageRouter）
    UPROPERTY(EditAnywhere) bool bSkippable = true;
};

UCLASS(BlueprintType)
class UDA_TutorialDefinition : public UPrimaryDataAsset  // = 模式B"下发配置"
{
    UPROPERTY(EditAnywhere) FGameplayTag TutorialId;       // "Tutorial.Newbie"
    UPROPERTY(EditAnywhere) TArray<FDOTutorialStep> Steps;
    UPROPERTY(EditAnywhere) TArray<FGameplayTag> PrerequisiteTutorials;
};

// 权威端：存于 ADOPlayerState（复制给客户端）
USTRUCT()
struct FDOTutorialProgress
{
    GENERATED_BODY()
    UPROPERTY(Replicated) TArray<FGameplayTag> CompletedTutorials;
    UPROPERTY(Replicated) FGameplayTag CurrentTutorialId, CurrentStepId;
};
```

## 4. 与项目已有系统的真实咬合点
| 引导步骤 | 完成信号来源（真实） |
|---|---|
| 移动教学 | 输入系统消费 `InputTag.Jump`/移动后广播的消息 |
| 释放技能 | `UDOAbilitySystemComponent` 技能激活时经消息总线广播 |
| 打开技能树 | UI 打开 `WBP_SkillTreeScreen` 广播 `Message.UI.SkillTree.Opened` |
| 升级一次技能 | 调已实现的 `Server_RequestAbilityLevel` 成功后，`OnAbilityLevelChanged` 委托触发 |

> 引导只认 `GameplayTag` 消息，不关心触发方。复杂条件由业务合成 Tag 后广播，引导逻辑零改动。

## 5. 核心流程（含权威端同步）
```text
Initialize()                     // LocalPlayer 创建时
  └─ 读 ADOPlayerState 复制来的 FDOTutorialProgress（=断点续引真值）
  └─ CurrentStepId 有效 → ResumeStep()（重高亮、重挂监听）
  └─ 否则等触发条件

StartTutorial(TutorialId)
  └─ 校验 PrerequisiteTutorials 都已完成
  └─ 载入 UDA_TutorialDefinition，定位首个未完成步骤
  └─ Server RPC：DOPlayerState->Server_SetCurrentStep(TutorialId, StepId)
  └─ EnterStep()

EnterStep(Step)
  └─ BroadcastMessage(Message.UI.Tutorial.StepChanged, {StepIndex,TotalSteps})
  └─ FocusWidgetTag 有效 → BroadcastMessage(Message.UI.Tutorial.Focus, {WidgetTag})
  └─ 注册 CompleteOnTags 监听（一次性）

OnCompleteTag(Tag)
  └─ Server RPC：DOPlayerState->Server_CompleteStep(StepId)  // 权威端记录
  └─ 本地存 SaveGame 断点（TutorialSave_<LocalPlayerId>，仅缓存/续引用）
  └─ ClearFocus → 下一步 / FinishTutorial()

FinishTutorial()
  └─ Server RPC 清 CurrentStep
  └─ BroadcastMessage(Message.UI.Tutorial.Available, false)  // 联动红点
```
- 每次 `EnterStep`/完成立即 `SaveGameToSlot`（本地断点缓存）+ Server RPC（权威真值）。
- `SaveVersion` 不匹配则丢弃旧档；断线重连以 `PlayerState` 复制的 `FDOTutorialProgress` 校准，本地旧档仅作兜底。

## 6. UI 焦点高亮（DOTutorialHighlightWidget）
- 全屏 `UUserWidget`，Root 为 `UCanvasPanel`，内含自定义 `SLeafWidget`。
- `OnPaint` 画四块半透明黑遮罩（上/下/左/右）留"洞" + 焦点区脉冲边框（用 `FSlateApplication::Get().GetCurrentTime()` 呼吸）。
- 驱动：C++ 直驱 `FocusWidget(UWidget*)` 读 `GetCachedGeometry()`；或监听 `Message.UI.Tutorial.Focus` 由蓝图按 `WidgetTag` 解析。
- `Tick` 中高亮态 `Invalidate(Paint)` 维持动画。纯 C++ 自绘，零美术依赖。

## 7. 异常与日志
- DataAsset 缺失 / 断点失效 → `UE_LOG(LogDragonOath, Error, ...)` + `DO_LOG_EXCEPTION`（见 `03_日志系统.md`）。
- 全部 `UE_LOG` 自动被日志系统落盘，业务零侵入。
