# 项目相关版 · 日志系统

> 落地于 DragonOath：在已有 `DOLogChannels` 下游加"持久化 + 分级过滤 + 堆栈捕获 + 异步落盘"。
> 子系统归属 `UGameInstanceSubsystem`（全局，合理）。UE 专属翻译：朋友资料的"6 级" = UE `ELogVerbosity`；"崩溃钩子" = `FCoreDelegates::OnHandleSystemError`。

## 0. 与朋友资料的能力对照
| 资料说法 | UE 落地 |
|---|---|
| 6 级分级（Debug/Info/Warn/Error/Fatal…） | `ELogVerbosity`（VeryVerbose/Verbose/Display/Log/Warning/Error/Fatal） |
| 全异步队列 0 卡主线程 | 生产者-消费者队列 + `FTicker` 后台落盘 |
| 本地滚动日志、自动清理 | 单文件 5MB 滚动 + 过期归档 |
| 全局异常 + 崩溃全捕获 | `FCoreDelegates::OnHandleSystemError` + `FPlatformStackWalk` 抓栈 |
| 结构化 JSON、后端解析 | 格式化行含 时间/级别/通道/玩家ID；错误日志可经 `FHttp` 上报 |
| 仅上传错误日志 | `Fatal`/`Error` 才进上传队列 |
| 线上动态管控级别 | 控制台变量 `DOTrace.MinVerbosity` 实时调 |
| 不重复造轮子 | 挂 `GLog` 输出设备，拦截现有 `UE_LOG`，业务零改动 |

## 1. 模块划分
```text
Source/DragonOath/Systems/Log/
  DOTraceSubsystem.h/.cpp      UGameInstanceSubsystem：总控、落盘调度、堆栈捕获
  DOTraceOutputDevice.h/.cpp   FOutputDevice 子类：拦截 GLog，过滤后入队
  DOTraceMacros.h              DO_LOG_EXCEPTION / DO_CAPTURE_STACK 便捷宏
```

## 2. 复用已有日志通道（不重造）
`Source/DragonOath/DOLogChannels.h` 已声明 `LogDragonOath`、`LogDragonOathAbilitySystem` 等。
本系统**不改**它们，而是把 `UDOTraceOutputDevice` 挂到 `GLog`：
```cpp
GLog->AddOutputDevice(DOTraceOutputDevice);  // 引擎/业务每条 UE_LOG 都会进入 Serialize
```
业务方继续 `UE_LOG(LogDragonOath, ...)` 写日志，自动落盘，**零侵入**。

## 3. 分级输出（贴 UE 语义）
- 复用 `ELogVerbosity`，映射资料 6 级：`Fatal`/`Error`(Error) / `Warning`(Warn) / `Display`+`Log`(Info) / `Verbose`+`VeryVerbose`(Debug/Trace)。
- `Serialize(V, Verbosity, Category, Time)` 按 `MinVerbosity` 过滤；`Fatal` 永远记录。
- 格式化：`[时间][级别][通道][PlayerId] 消息`。
- 运行时用控制台变量 `DOTrace.MinVerbosity`（0~6）调节最小落盘级别（资料"线上动态管控"）。

## 4. 线程安全落盘（生产者-消费者）
`Serialize` 可能来自任意线程，**不能**在回调里写文件：
```text
EnqueueLine(Line)
  └─ FScopeLock(PendingLock)
       PendingLines.Add(Line)         // 待落盘队列
       RingBuffer.Add(Line)           // 内存环形缓冲（上限 512，超则丢最旧）

FTicker 每帧 TickFlush()
  └─ FScopeLock(PendingLock); Batch = MoveTemp(PendingLines)
  └─ 文件超限 → RotateFile()（归档当前、开新文件）
  └─ FFileHelper::SaveStringToFile(Batch, Append, UTF8无BOM)

落盘路径：Saved/Logs/Trace/DragonOath_<SessionGuid>.log   // 单会话一文件，便于归因
```
- 环形缓冲供调试 UI（`DOTrace.Dump`）零 IO 读最近 N 条。
- 单文件上限 5MB，超限滚动（资料"自动分割"）。
- 退出时 `FlushToFile()` 保剩余日志不丢（资料"游戏退出自动落盘"）。

## 5. 异常堆栈捕获（真实 API）
```cpp
static FString CaptureStackTrace(int32 SkipFrames = 2, int32 MaxFrames = 32)
{
    TArray<uint64> BT; BT.SetNum(MaxFrames);
    const int32 N = FPlatformStackWalk::CaptureStackBackTrace(BT.GetData(), MaxFrames, nullptr, SkipFrames);
    BT.SetNum(N);
    FString Out;
    for (int32 i = 0; i < N; ++i)
    {
        ANSICHAR Sym[1024], File[512]; int32 Line = 0;
        FPlatformStackWalk::Symbolize(BT[i], Sym, sizeof(Sym), File, sizeof(File), &Line);
        Out += FString::Printf(TEXT("#%d 0x%016llX %s (%s:%d)\n"), i, BT[i], ANSI_TO_TCHAR(Sym), ANSI_TO_TCHAR(File), Line);
    }
    return Out;
}
```
两类入口（资料"两类捕获"）：
1. **业务异常** `LogException(Ctx, Detail)`：主动调用，自动抓栈 + 落盘 + `UE_LOG(Error)`。
2. **引擎崩溃** `FCoreDelegates::OnHandleSystemError.AddUObject(...)`：回调里抓栈 → 入队 → **同步** `FlushToFile()`，尽力保住现场（资料"黑匣子"）。

## 6. 错误上报（可选，资料"仅上传错误"）
- `Fatal`/`Error` 行额外入 `UploadQueue`，由 `FTicker` 经 `FHttpModule` 异步发往后端日志服务（DragonOath 后端/监控平台），省流量只传错误。

## 7. 便捷宏
```cpp
#define DO_CAPTURE_STACK()  UDOTraceSubsystem::CaptureStackTrace(1, 32)
#define DO_LOG_EXCEPTION(Ctx, Fmt, ...) \
    if (UDOTraceSubsystem* T = UDOTraceSubsystem::Get()) \
        T->LogException(TEXT(#Ctx), FString::Printf(TEXT(Fmt), ##__VA_ARGS__))
```

## 8. 与引导/红点的对接
- 三系统 `UE_LOG` 自动落盘，无需额外接线上。
- 关键异常（DataAsset 缺失、红点成环、断点失效）→ `DO_LOG_EXCEPTION` 抓栈。
- 调试时在 `Saved/Logs/Trace/*.log` 按 `LogDragonOath*` 通道过滤。

## 9. 控制台命令
| 命令 | 作用 |
|---|---|
| `DOTrace.MinVerbosity <0-6>` | 运行时调最小落盘级别（线上动态管控） |
| `DOTrace.Dump` | 打印内存环形缓冲最近 N 条 |
| `DOTrace.Flush` | 立即落盘 |
