# 项目相关版 · 系统间交互设计

> 贴 DragonOath：把朋友资料的"服务端权威"翻译成 UE 权威端（DS/Listen Server + `PlayerState` 复制 + Server RPC）。

## 1. 拓扑（含 UE 权威端）
```text
                ┌──────── UE 权威端（DS / Listen Server 的 C++ 逻辑）────────┐
                │ ADOPlayerState: 引导进度 + 红点真值(已读/数量)  日志上报接收 │
                └───────┬──────────────────────┬────────────────────┬───────┘
           属性复制下发   │         OnRep 推送变更   │           接收错误上报
                        ▼                        ▼                    ▼
业务系统（技能/邮件/任务/输入）
   │ SetNodeActive/SetNodeCount/MarkNodeDirty   │ BroadcastMessage(完成Tag)
   ▼                                            ▼
DORedDotSubsystem(LocalPlayer)      DOTutorialSubsystem(LocalPlayer)
   │ 引导进行中→SetNodeActive(Tutorial,true)     │ 进入/完成步骤
   └──────────────┐              ┌──────────────┘
                  ▼              ▼
        GameplayMessageRouter（Message.UI.*）
                  │
                  ▼
           UI（UMG / Common UI）只监听、只展示

所有系统 ──UE_LOG──▶ DOTraceSubsystem(GameInstance) ──▶ 文件 + 环形缓冲 + 崩溃抓栈 ──错误上报──▶ 权威端/监控
```

## 2. 权威端翻译对照表（资料协议 → UE）
| 资料协议 | UE 落地 |
|---|---|
| `C2S_GetAllRedPoint` | 登录读 `PlayerState` 已复制的 `FDORedDotState`（属性复制自带） |
| `S2C_RedPointData` | `PlayerState` 属性复制下发（无需手写） |
| `S2C_PushRedPointChange` | 服务端改 `PlayerState` 属性 → `OnRep` 触发 |
| `C2S_MarkRedPointRead` | 客户端 `Server RPC` → 改 `PlayerState.ReadNodes` |
| 引导配置下发（模式B） | `DA_TutorialDefinition` 本地资产即"下发配置" |
| 引导进度同步（模式B） | `Server RPC` 写 `PlayerState.FDOTutorialProgress` + 复制 |

## 3. 引导 ↔ 红点（弱耦合，都在 LocalPlayer 上）
```text
DOTutorialSubsystem::StartTutorial()
  └─ RedDot = GetLocalPlayer()->GetSubsystem<UDORedDotSubsystem>()
  └─ RedDot->SetNodeActive("RedDot.Root.Tutorial", true)   → 入口红点点亮
DOTutorialSubsystem::FinishTutorial()
  └─ RedDot->SetNodeActive("RedDot.Root.Tutorial", false)  → 入口红点清除
```
解耦原则：引导不拥有红点逻辑，只表达"我有引导要做"；删掉这两行即解耦。

## 4. 真实数据源 → 红点
```text
技能可升级：
  UDOAbilitySystemComponent::OnAbilityLevelChanged 委托触发（已实现）
    └─ 未来技能点属性变化 / 等级<Max → RedDot->MarkNodeDirty("RedDot.Root.Skill.Upgradable")
    └─ 0.2s 后批量重算 → 父级聚合 → BroadcastMessage(Message.UI.RedDot.Changed)

邮件/任务（未来系统）：PlayerState 复制 OnRep 或状态变化时 → SetNodeCount / MarkNodeDirty
```

## 5. 引导的消息驱动完成（真实链路）
```text
玩家完成"释放技能"步骤：
  技能激活 → DOAbilitySystemComponent 经消息总线广播 "Message.Combat.Ability.Activated"
    └─ DOTutorialSubsystem 监听 CompleteOnTags 命中 → Server RPC 记录 → 进入下一步
```
引导只认 Tag，不关心触发方；复杂条件由业务合成 Tag 后广播。

## 6. 日志：横切所有系统
- 三系统 `UE_LOG(LogDragonOath, ...)` 自动落盘（经 `GLog` 输出设备），零侵入。
- 关键异常用 `DO_LOG_EXCEPTION` 抓栈；崩溃由 `OnHandleSystemError` 兜底。
- 错误日志可经 `FHttp` 上报 UE 权威端/监控平台（资料"仅上传错误"）。

## 7. 共享消息 Payload（Source/DragonOath/Systems/DOMessageTypes.h）
| 频道 | Payload |
|---|---|
| `Message.UI.RedDot.Changed` | `FDORedDotChangedMessage{NodeId, Count}` |
| `Message.UI.Tutorial.StepChanged` | `FDOTutorialStepMessage{TutorialId, StepId, StepIndex, TotalSteps, bCompleted}` |
| `Message.UI.Tutorial.Focus` / `ClearFocus` | `FDOTutorialFocusMessage{WidgetTag, bShow, ScreenPosition, ScreenSize, Padding}` |
| `Message.UI.Tutorial.Available` | 空 / 带 TutorialId |

> Payload 只描述"已发生的事实"，不承载"请求服务器"（见 `Docs/04_Local_Message_Bus.md`）。

## 8. 联机验证（实现后）
- PIE Listen Server + 1 Client：客户端引导步骤 → 遮罩出现 → 完成（Server RPC 写 PlayerState）→ 另一 Client 经复制同步 → 红点清除（各客户端只算自己）。
- 红点：制造可升级技能 → 入口红点点亮 → 升级后清除（父级自动灭）；点红点本地立即灭 → Server 改 PlayerState.ReadNodes → 多端同步灭。
- 日志：主动 `DO_LOG_EXCEPTION` → 查 `Saved/Logs/Trace/*.log` 含堆栈；`ensure` 触发 `OnHandleSystemError` → 确认落盘。
- 断点续引：引导到第三步退出重进 → 以 `PlayerState.FDOTutorialProgress` 复制值校准，从第三步继续。
